# 15-1. 선형 자료구조 — 실습

> **참고** 이 실습은 제출물이 아니라, 배열과 연결 리스트의 내부 동작을 직접 구현하고 측정해보는 과정입니다.

## 실습 1 — 동적 배열의 재할당 관찰하기

- [ ] 빈 리스트에 원소를 100,000개 추가하면서 매 단계의 실제 할당 용량을 기록하세요(Python이면 `sys.getsizeof(lst)`, Java면 리플렉션으로 내부 `elementData.length`).
- [ ] 용량이 바뀌는 지점의 원소 개수만 뽑아 나열하고, 이전 용량 대비 몇 배로 늘었는지 계산하세요.
- [ ] 성장 배수를 2배로 고정한 동적 배열과 매번 1칸씩만 늘리는 동적 배열을 직접 구현해, 원소 50,000개 추가에 걸리는 시간을 각각 측정하고 비교하세요.

## 실습 2 — 배열 vs 연결 리스트 실측

- [ ] 정수 100,000개를 담은 배열과, 직접 구현한 단일 연결 리스트를 각각 만드세요.
- [ ] 두 구조에서 전체 원소의 합을 구하는 순회를 각 100회 반복해 시간을 측정하고, 같은 $O(n)$인데도 차이가 나는 배수를 기록하세요.
- [ ] 두 구조의 맨 앞에 원소를 10,000번 삽입하는 시간을 측정하고, 이번에는 어느 쪽이 빠른지 확인하세요.

## 실습 3 — 원형 버퍼로 큐 만들기

- [ ] 고정 크기 배열 하나와 `front`, `rear`, `count` 세 변수만 사용해 `enqueue`/`dequeue`를 $O(1)$로 구현하세요. 인덱스 증가는 반드시 `(i + 1) % capacity`로 처리합니다.
- [ ] 용량 5인 버퍼에 8번 넣고 5번 빼는 시나리오를 실행하며, 매 연산 후 `front`, `rear`, 내부 배열 상태를 출력해 되감기는 순간을 눈으로 확인하세요.
- [ ] "가득 참"과 "비었음"을 `count` 없이 구분하려면 어떤 규칙이 필요한지 코드로 구현해보세요.
- [ ] 같은 큐를 `list.pop(0)` 방식으로도 만들어, 원소 100,000개를 넣고 빼는 시간을 원형 버퍼 버전과 비교하세요.

## 실습 4 — 이중 연결 리스트로 LRU 캐시 만들기

- [ ] 이중 연결 리스트 노드를 값으로 갖는 해시 맵을 만들고, `get(key)` 호출 시 해당 노드를 리스트 맨 앞으로 옮기는 코드를 작성하세요.
- [ ] 용량 초과 시 리스트 맨 뒤 노드를 제거하는 `put(key, value)`를 구현하고, 노드 이동과 제거가 각각 포인터 몇 개를 고쳐 쓰는지 세어보세요.
- [ ] 같은 기능을 배열만으로 구현한 뒤, 원소 10,000개 기준으로 `get` 100,000회 실행 시간을 두 버전에서 측정해 비교하세요.

> **팁** 실습 2와 3의 측정값은 절대 시간보다 **비율**이 중요합니다. 실행 환경마다 숫자는 달라져도, 캐시 지역성 때문에 배열 순회가 연결 리스트 순회보다 빠르다는 방향은 거의 항상 재현됩니다.
