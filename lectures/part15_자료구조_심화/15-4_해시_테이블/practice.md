# 15-4. 해시 테이블 — 실습

> **참고** 이 실습은 제출물이 아니라, 해시 테이블을 직접 만들어 평균 $O(1)$이 무너지는 조건을 손으로 재현해보는 과정입니다.

## 실습 1 — 해시 분포 측정하기

- [ ] 버킷 1,000개를 가정하고, 영어 단어 10,000개(사전 파일이나 무작위 문자열)를 `hash(key) % 1000`으로 분배한 뒤 버킷별 원소 수를 세어보세요.
- [ ] 가장 원소가 많은 버킷과 가장 적은 버킷의 개수 차이를 구하고, 이상적인 균등 분포값(10)과 얼마나 벗어나는지 계산하세요.
- [ ] 해시 함수를 `문자열 길이 % 1000`처럼 나쁜 함수로 바꿔 같은 측정을 하고, 두 분포의 히스토그램을 비교하세요.
- [ ] 버킷 수를 1,024(2의 거듭제곱)와 1,009(소수)로 바꿔가며 분포가 달라지는지 확인하세요.

## 실습 2 — 체이닝 해시 테이블 직접 구현하기

- [ ] 배열 + 각 칸의 리스트 구조로 `put`, `get`, `remove`를 구현하세요. 언어 내장 맵은 쓰지 않습니다.
- [ ] `get` 호출 시 비교 연산 횟수를 반환하도록 만들고, 원소 1,000개를 넣은 뒤 평균 비교 횟수를 측정하세요.
- [ ] 리사이징 없이 원소를 10,000개까지 늘려가며 평균 비교 횟수를 기록하고, 적재율 $\alpha$와 거의 비례하는지 확인하세요.
- [ ] 적재율 0.75에서 버킷을 2배로 늘리고 전체를 재해싱하는 코드를 추가한 뒤, 같은 측정을 반복해 평균 비교 횟수가 상수로 유지되는지 확인하세요.

## 실습 3 — 개방 주소법과 툼스톤

- [ ] 선형 탐사 방식의 개방 주소법 해시 테이블을 구현하세요.
- [ ] 충돌이 일어나도록 값을 넣은 뒤, 중간 원소를 **툼스톤 없이** 삭제하고 그 뒤쪽 원소를 `get`으로 찾아보세요. 실패하는 것을 직접 확인합니다.
- [ ] 툼스톤 표식을 도입해 같은 시나리오가 정상 동작하도록 고치세요.
- [ ] 적재율을 0.5부터 0.95까지 0.05씩 올려가며 실패 탐색의 평균 탐사 횟수를 측정하고, 0.7 부근부터 곡선이 치솟는지 그래프로 확인하세요.

## 실습 4 — equals-hashCode 계약 깨뜨려보기

- [ ] 필드가 두 개인 클래스를 만들고, 두 필드가 같으면 같다고 판정하되 `hashCode`(또는 `__hash__`)는 **필드 하나만** 사용하도록 구현하세요. 넣은 값을 다시 찾을 수 있는지 확인합니다.
- [ ] 이번에는 `hashCode`를 항상 같은 상수로 반환하도록 바꾸고, 원소 10,000개를 넣은 뒤 조회 시간이 얼마나 느려지는지 측정하세요.
- [ ] 맵의 키로 쓴 객체의 필드를 넣은 뒤에 변경하고, `get`이 실패하는 것을 재현한 다음 왜 그런지 버킷 인덱스로 설명하세요.

> **팁** 실습 4의 두 번째 항목이 해시 충돌 DoS 공격의 축소판입니다. 모든 키가 한 버킷으로 몰리면 해시 테이블은 그냥 연결 리스트가 되며, Java 8의 트리화가 막으려던 것이 바로 이 상황입니다.
