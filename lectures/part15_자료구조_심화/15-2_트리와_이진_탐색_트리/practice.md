# 15-2. 트리와 이진 탐색 트리 — 실습

> **참고** 이 실습은 제출물이 아니라, 트리를 직접 만들고 무너뜨려보며 균형의 필요성을 체감하는 과정입니다.

## 실습 1 — 순회 네 가지 직접 구현하기

- [ ] 값 `[50, 30, 70, 20, 40, 60, 80]`을 이 순서대로 삽입해 이진 탐색 트리를 만드세요.
- [ ] 전위·중위·후위 순회를 재귀로 구현해 각각의 출력 수열을 종이에 먼저 적고, 코드 실행 결과와 일치하는지 확인하세요.
- [ ] 큐를 사용해 레벨 순회를 구현하고, 각 노드를 방문할 때 그 노드의 깊이도 함께 출력하세요.
- [ ] 중위 순회 결과가 오름차순으로 정렬되어 있는지 확인하고, 그 이유를 BST 불변식으로 설명해보세요.

## 실습 2 — 삭제의 세 경우 처리하기

- [ ] 실습 1의 트리에서 잎 노드 `20`을 삭제하고, 삭제 후 중위 순회가 여전히 정렬 상태인지 확인하세요.
- [ ] 자식이 하나인 노드를 만들기 위해 `40`을 먼저 지운 뒤 `30`을 삭제하고, 어떤 노드가 그 자리로 올라오는지 출력하세요.
- [ ] 자식이 둘인 루트 `50`을 삭제하는 코드를 작성하되, 중위 후속자(오른쪽 서브트리의 최솟값)를 찾아 값을 덮어쓰는 방식으로 구현하세요.
- [ ] 중위 후속자 대신 중위 선행자(왼쪽 서브트리의 최댓값)를 쓰도록 바꿔보고, 두 결과 트리의 모양이 어떻게 달라지는지 비교하세요.

## 실습 3 — 편향 트리 재현하고 측정하기

- [ ] 1부터 10,000까지의 정수를 **오름차순 그대로** BST에 삽입한 뒤, 완성된 트리의 높이를 재귀로 계산해 출력하세요.
- [ ] 같은 10,000개의 값을 무작위로 섞어 삽입하고 높이를 다시 계산해, 두 높이를 비교하세요.
- [ ] 두 트리에서 값 `9999`를 탐색할 때 비교 연산이 각각 몇 회 일어나는지 카운터를 넣어 세어보세요.
- [ ] 오름차순 삽입 트리에서 재귀 순회를 실행해 스택 오버플로가 발생하는 입력 크기를 찾아보세요.

## 실습 4 — 표준 라이브러리의 균형 트리 확인하기

- [ ] Java `TreeMap` 또는 C++ `std::map`에 1부터 100만까지의 키를 오름차순으로 삽입하고 총 소요 시간을 측정하세요.
- [ ] 같은 데이터를 실습 3에서 만든 직접 구현 BST에 삽입해보고(가능한 크기까지), 두 결과의 차이를 기록하세요.
- [ ] `TreeMap.subMap()`이나 `std::map`의 `lower_bound`로 범위 질의를 실행하고, 같은 질의를 해시 맵으로 하려면 무엇이 필요할지 코드로 적어보세요.

> **팁** 실습 3이 이 강의의 핵심입니다. 자료구조의 성능이 데이터 자체가 아니라 **데이터가 도착한 순서**에 좌우된다는 사실을 숫자로 확인하면, 왜 실무에서 직접 만든 BST 대신 표준 라이브러리의 균형 트리를 쓰는지 납득할 수 있습니다.
