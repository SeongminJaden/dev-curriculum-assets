# 15-3. 힙과 우선순위 큐 — 실습

> **참고** 이 실습은 제출물이 아니라, 힙을 배열 하나로 직접 구현하고 복잡도 주장을 손으로 검증하는 과정입니다.

## 실습 1 — 인덱스 산술 확인하기

- [ ] 배열 `[1, 3, 6, 5, 9, 8]`을 최소 힙으로 보고, 각 인덱스의 부모·왼쪽 자식·오른쪽 자식 인덱스를 공식으로 계산해 표로 적으세요.
- [ ] 계산한 부모-자식 쌍마다 힙 성질(부모 ≤ 자식)이 성립하는지 하나씩 확인하세요.
- [ ] 배열 `[1, 5, 3, 9, 6, 8]`에 대해 같은 검사를 수행하고, 성질을 어기는 쌍이 있다면 어느 인덱스인지 찾아내세요.

## 실습 2 — sift-up과 sift-down 직접 구현하기

- [ ] 파이썬 리스트(또는 배열) 하나만 사용해 `push`(sift-up)와 `pop_min`(sift-down)을 구현하세요. 포인터나 노드 클래스는 쓰지 않습니다.
- [ ] `push`와 `pop_min`에 교환 횟수 카운터를 넣고, 원소 1,024개를 넣고 뺄 때 1회당 교환 횟수가 10회(= $\log_2 1024$)를 넘지 않는지 확인하세요.
- [ ] 최소 힙 코드에서 부등호만 뒤집어 최대 힙을 만들고, 두 힙에 같은 데이터를 넣어 pop 순서가 정확히 반대인지 확인하세요.

## 실습 3 — heapify가 O(n)임을 측정으로 확인하기

- [ ] 무작위 배열을 `push`를 $n$번 반복해 힙으로 만드는 함수와, 마지막 내부 노드부터 sift-down하는 `heapify` 함수를 각각 구현하세요.
- [ ] 두 함수의 총 교환 횟수를 $n$ = 1,000 / 10,000 / 100,000에 대해 각각 측정해 표로 정리하세요.
- [ ] $n$이 10배 늘 때 각 함수의 교환 횟수가 몇 배로 늘어나는지 계산하고, 한쪽만 정확히 10배 근처인지 확인하세요.
- [ ] 완성된 heapify 위에 힙 정렬을 구현하고, 언어 표준 정렬과 실행 시간을 비교하세요.

## 실습 4 — 상위 k개 추출 비교하기

- [ ] 무작위 정수 100만 개에서 가장 큰 100개를 구하는 코드를 두 가지로 작성하세요. (1) 전체 정렬 후 앞 100개 자르기, (2) 크기 100짜리 최소 힙을 유지하며 한 번만 순회하기.
- [ ] 두 방식의 실행 시간과 최대 메모리 사용량을 측정해 비교하세요.
- [ ] $k$를 100 → 10,000 → 100,000으로 늘려가며, 힙 방식의 이점이 사라지기 시작하는 지점을 찾아보세요.

> **팁** 실습 3이 이 강의의 하이라이트입니다. $O(n)$과 $O(n \log n)$의 차이는 실행 시간보다 **교환 횟수**로 재는 편이 훨씬 선명하게 드러납니다. 환경 잡음이 섞이지 않기 때문입니다.
