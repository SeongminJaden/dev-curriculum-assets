# 6-3. API 설계 패턴 — 실습

> **참고** 정답을 제출하는 과제가 아니라, REST·GraphQL·gRPC가 겨냥한 문제를 스스로 구분해보는 실습입니다.

## 체크리스트

- [ ] 자신이 최근 사용해본 서비스의 화면 하나(예: 상품 상세 페이지, 프로필 화면)를 골라, 그 화면을 그리는 데 필요한 데이터 종류를 모두 나열해보세요.
- [ ] 위 화면을 REST로 구현한다면 엔드포인트를 몇 번 호출해야 할지, 오버페칭이나 언더페칭이 발생할 만한 지점이 있는지 찾아보세요.
- [ ] 같은 화면을 GraphQL 쿼리로 표현한다면 어떤 필드를 요청할지 쿼리 형태(중첩 구조)로 직접 적어보세요.
- [ ] 이 서비스 내부에 여러 마이크로서비스(주문, 결제, 재고 등)가 있다고 가정하고, 그중 gRPC로 통신하는 것이 유리한 구간과 그 이유를 설명해보세요.
- [ ] 본문 5절의 배달 앱 표를 참고해, 자신이 고른 서비스에 대해 통신 구간별로 REST·GraphQL·gRPC 중 무엇을 쓸지 동일한 형식의 표를 작성해보세요.
- [ ] 만약 이 서비스의 공개 API에 GraphQL 대신 REST를 계속 써야 한다는 반대 의견이 나온다면, 어떤 근거로 반박하거나 동의할지 한 문단으로 정리해보세요.
