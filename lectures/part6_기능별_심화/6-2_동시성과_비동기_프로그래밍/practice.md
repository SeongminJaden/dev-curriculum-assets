# 6-2. 동시성과 비동기 프로그래밍 — 실습

> **참고** 정답을 제출하는 과제가 아니라, 동시성과 병렬성, 컨텍스트 스위칭, 이벤트 루프 개념을 스스로 적용해보는 실습입니다.

## 체크리스트

- [ ] 최근 사용해본 서비스에서 "동시성"과 "병렬성"이 각각 어디에 해당하는지 예시를 하나씩 찾아 설명해보세요. (예: 서버가 여러 요청을 번갈아 처리하는 것 vs 여러 코어가 이미지 여러 장을 동시에 인코딩하는 것)
- [ ] 자신이 아는 프로그램(웹 서버, 게임, 채팅 앱 등) 하나를 골라, 그 프로그램이 처리하는 대표적인 작업이 CPU 바운드인지 I/O 바운드인지 근거를 들어 판단해보세요.
- [ ] 위에서 고른 작업을 멀티스레딩 방식으로 처리한다면 스레드 수를 얼마나 늘렸을 때 컨텍스트 스위칭 비용이 이득보다 커질지, 그렇게 판단한 이유를 적어보세요.
- [ ] 같은 작업을 이벤트 루프 방식으로 처리한다면 어떤 상황에서 루프 전체가 지연될 위험이 있는지 찾아보고, 이를 피하려면 어떻게 설계해야 할지 적어보세요.
- [ ] "채팅 서버" 표(본문 4절)를 참고해, 자신이 고른 서비스에 대해 동일한 형식의 비교표를 직접 작성해보세요.
- [ ] 자신의 결론을 바탕으로, 이 서비스를 새로 설계한다면 멀티스레딩과 이벤트 루프 중 무엇을 선택할지, 그리고 CPU 바운드 작업이 섞여 있다면 어떻게 분리해서 처리할지 한 문단으로 정리해보세요.
