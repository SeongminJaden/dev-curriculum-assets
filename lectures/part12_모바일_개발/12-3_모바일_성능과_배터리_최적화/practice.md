# 12-3 실습 — 모바일 성능과 배터리 최적화

> **참고** 제출물이 아니라, 라디오 전력 상태와 배칭 효과, 배터리 소모 패턴을 직접 진단해보는 실습입니다.

## 실습 1. 배터리를 많이 먹는 앱 진단하기

- [ ] 스마트폰의 배터리 사용량 화면(설정 → 배터리)을 열어, 배터리를 많이 쓴 앱 상위 3개를 확인해보세요.
- [ ] 각 앱이 강의에서 다룬 패턴(실시간 위치 추적, 짧은 주기의 폴링, 백그라운드 렌더링) 중 어디에 해당할 것 같은지 추측해보세요.
- [ ] 그렇게 추측한 근거를 앱의 실제 사용 방식(예: "이 앱은 배달 위치를 실시간으로 보여준다")과 연결해 한 문단으로 적어보세요.

## 실습 2. 배칭 효과 직접 계산해보기

- [ ] 여러분이 만드는 앱이 10초마다 서버에 작은 상태 업데이트를 보낸다고 가정해보세요.
- [ ] 요청 1번당 라디오가 고전력 상태(전송 + 테일 타임)를 유지하는 시간을 임의로 정하고(예: 5초), 1분 동안 몇 번의 고전력 구간이 발생하는지 계산해보세요.
- [ ] 같은 데이터를 30초 주기로 배칭해서 보낸다면 고전력 구간이 몇 번으로 줄어드는지 비교해보세요.
- [ ] 배칭 주기를 더 늘렸을 때(예: 60초) 사용자 경험(데이터 신선도)과 배터리 절약 사이에 어떤 트레이드오프가 생기는지 적어보세요.
