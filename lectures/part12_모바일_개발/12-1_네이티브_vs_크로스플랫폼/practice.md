# 12-1 실습 — 네이티브 vs 크로스플랫폼

> **참고** 제출물이 아니라, 오늘 배운 세 가지 렌더링 아키텍처(네이티브, React Native 브리지, Flutter 자체 엔진)를 직접 비교하며 정리해보는 실습입니다.

## 실습 1. 익숙한 앱 하나를 골라 아키텍처 추측해보기

- [ ] 평소 자주 쓰는 모바일 앱을 하나 골라주세요.
- [ ] 그 앱의 버튼 애니메이션, 스크롤 느낌, 화면 전환 방식이 iOS/Android 기본 앱(설정, 연락처 등)과 얼마나 비슷한지 관찰해보세요.
- [ ] 관찰한 내용을 바탕으로 이 앱이 네이티브인지, React Native인지, Flutter인지 추측하고 그 근거를 한 문단으로 적어보세요.
- [ ] (선택) 실제로 어떤 기술로 만들어졌는지 검색해서 추측이 맞았는지 확인해보세요.

## 실습 2. 트레이드오프 표 채우기

- [ ] 아래 표의 빈칸을 강의 내용을 참고해 채워보세요.

| 비교 기준 | 네이티브 | React Native | Flutter |
|---|---|---|---|
| UI를 그리는 주체 | | | |
| 코드가 공유되는 범위 | | | |
| OS 디자인 변경에 대한 자동 반영 여부 | | | |

- [ ] 표를 채운 뒤, 만약 여러분이 팀 리더라면 다음 두 앱 중 어떤 프레임워크를 선택할지, 그리고 그 이유를 각각 적어보세요.
  - [ ] 은행 앱처럼 각 플랫폼의 사용자 경험 규범(예: iOS의 스와이프 뒤로가기)을 정확히 따라야 하는 앱
  - [ ] 브랜드 디자인을 iOS와 Android에서 완전히 동일하게 유지하고 싶은 커머스 앱
