# 12-2 실습 — 모바일 앱 생명주기와 상태 관리

> **참고** 제출물이 아니라, iOS/Android 생명주기 콜백과 상태 저장 시나리오를 직접 정리해보는 실습입니다.

## 실습 1. 내가 쓰는 앱에서 상태 손실 찾아보기

- [ ] 평소 자주 쓰는 앱 하나를 골라, 무언가를 입력하거나 진행하던 중(예: 글쓰기, 장바구니 담기, 지도 검색)에 홈 버튼을 눌러 백그라운드로 보내주세요.
- [ ] 카메라나 게임처럼 메모리를 많이 쓰는 다른 앱을 몇 개 실행해보세요.
- [ ] 처음 앱으로 다시 돌아와 진행 중이던 상태가 그대로 남아 있는지 확인해보세요.
- [ ] 상태가 그대로였다면 어떤 메커니즘이 동작했을지, 사라졌다면 어느 시점에 저장 로직이 빠졌을지 추측해서 적어보세요.

## 실습 3. 상태 저장 설계해보기

- [ ] 여러분이 메모 앱을 만든다고 가정해보세요. 사용자가 메모를 작성하던 중 앱이 백그라운드로 전환되었다가, OS에 의해 프로세스가 종료된 뒤 다시 실행되는 상황을 가정합니다.
- [ ] 어느 시점에 어떤 데이터를(임시 저장 vs 영구 저장) 저장해야 할지 한 문단으로 설계해보세요.
- [ ] Android라면 `ViewModel`만으로 충분한지, `SavedStateHandle`이나 Room 같은 영구 저장소가 추가로 필요한지 판단해보세요.

## 생각해볼 질문

- [ ] `applicationWillTerminate`나 `onDestroy`에 중요한 저장 로직을 넣으면 안 되는 이유를 한 문장으로 설명해보세요.
- [ ] 화면 회전으로 인한 상태 손실과 프로세스 종료로 인한 상태 손실은 왜 서로 다른 해법이 필요할까요?
