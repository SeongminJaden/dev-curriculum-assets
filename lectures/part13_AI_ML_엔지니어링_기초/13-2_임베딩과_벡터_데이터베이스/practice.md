# 13-2 실습 — 임베딩과 벡터 데이터베이스

> **참고** 제출물이 아니라, 오늘 배운 임베딩·코사인 유사도·ANN 검색 개념을 스스로 계산하고 적용해보는 실습입니다.

## 실습 1. 코사인 유사도 손으로 계산해보기

- [ ] 2차원 벡터 $A=(3,1)$, $B=(1,3)$을 준비해주세요.
- [ ] $A \cdot B$(내적)를 직접 계산해보세요.
- [ ] $\|A\|$, $\|B\|$(각 벡터의 크기)를 직접 계산해보세요.
- [ ] $\cos\theta = \dfrac{A \cdot B}{\|A\|\,\|B\|}$ 값을 계산하고, 이 값이 두 벡터가 얼마나 비슷한 방향인지를 어떻게 나타내는지 한 문장으로 적어보세요.

## 실습 2. 동의어 검색 실패 사례 찾아보기

- [ ] 평소 쓰는 검색 서비스(쇼핑몰, 문서 검색 등)에서 같은 의미인데 다른 단어를 썼을 때 검색이 되지 않는 사례를 하나 찾아보세요.
- [ ] 이 사례가 임베딩 기반 검색으로 바뀐다면 왜 해결될 수 있는지 설명해보세요.

## 실습 3. 브루트포스와 ANN 비교해보기

- [ ] 저장된 벡터가 100만 개라고 가정하고, 질의 한 번에 브루트포스 탐색이 몇 번의 비교 연산을 해야 하는지 적어보세요.
- [ ] ANN 검색이 이 문제를 완화하기 위해 어떤 대가(정확도)를 지불하는지 한 문장으로 설명해보세요.
