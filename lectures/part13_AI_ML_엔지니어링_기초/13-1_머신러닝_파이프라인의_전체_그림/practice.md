# 13-1 실습 — 머신러닝 파이프라인의 전체 그림

> **참고** 제출물이 아니라, 오늘 배운 파이프라인 6단계와 데이터 드리프트 개념을 스스로 적용해보는 실습입니다.

## 실습 1. 나만의 서비스로 파이프라인 그려보기

- [ ] 평소 사용하는 앱이나 서비스 중 머신러닝이 쓰이고 있을 법한 기능을 하나 골라주세요. (예: 음악 추천, 스팸 필터, 이미지 자동 태그)
- [ ] 그 기능이 아래 6단계 각각에서 어떤 데이터를 다루고 어떤 일을 할지 한 문장씩 적어보세요.
  - [ ] 데이터 수집
  - [ ] 전처리
  - [ ] 학습
  - [ ] 평가
  - [ ] 서빙
  - [ ] 모니터링

## 실습 2. 학습-서빙 스큐 찾아보기

- [ ] 실습 1에서 고른 기능의 전처리 단계를 하나 정해주세요. (예: 텍스트를 소문자로 변환, 이미지 크기를 정규화)
- [ ] 만약 이 전처리를 학습 코드에서는 적용하고 서빙 코드에서는 빠뜨렸다면 어떤 문제가 생길지 상상해서 적어보세요.
- [ ] 이 문제를 코드 리뷰나 테스트로 미리 잡아내려면 어떤 장치가 필요할지 생각해보세요.

## 실습 3. 데이터 드리프트 시나리오 만들기

- [ ] 실습 1에서 고른 서비스에서, 시간이 지나며 입력 데이터의 분포가 달라질 수 있는 상황을 하나 상상해보세요. (예: 신규 사용자 연령대 변화, 계절에 따른 상품 트렌드 변화)
- [ ] 이 변화가 입력 분포 드리프트(covariate drift)인지, 개념 드리프트(concept drift)인지 구분해보세요.
- [ ] 이 드리프트를 모니터링 단계에서 어떻게 감지할 수 있을지 한 문장으로 적어보세요.
