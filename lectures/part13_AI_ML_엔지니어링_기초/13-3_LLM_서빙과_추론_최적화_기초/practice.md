# 13-3 실습 — LLM 서빙과 추론 최적화 기초

> **참고** 제출물이 아니라, 오늘 배운 메모리 계산·KV 캐시·양자화·배치 추론 개념을 스스로 계산하고 적용해보는 실습입니다.

## 실습 1. 모델 메모리 사용량 직접 계산해보기

- [ ] 파라미터 130억 개(13B)인 모델을 가정해주세요.
- [ ] 이 모델을 FP32(4바이트)로 저장할 때 필요한 메모리(GB)를 계산해보세요.
- [ ] 같은 모델을 FP16(2바이트)로 저장할 때 필요한 메모리를 계산해보세요.
- [ ] 같은 모델을 INT4(0.5바이트)로 저장할 때 필요한 메모리를 계산해보세요.
- [ ] 세 값을 비교하고, 정밀도를 낮출 때마다 메모리가 어떻게 줄어드는지 한 문장으로 정리해보세요.

## 실습 2. KV 캐시 없이 생성한다면

- [ ] KV 캐시가 없다고 가정하고, 10번째 토큰을 생성할 때 몇 번째 토큰까지의 키·값을 다시 계산해야 하는지 적어보세요.
- [ ] 문장이 길어질수록(예: 10토큰 → 1000토큰) 이 재계산 비용이 어떻게 달라질지 설명해보세요.

## 실습 3. 양자화의 트레이드오프 정리하기

- [ ] 실습 1에서 계산한 메모리 차이를 참고해, 서비스 운영자가 INT4 양자화를 선택할 때 얻는 이득과 감수해야 하는 위험을 각각 한 문장씩 적어보세요.
