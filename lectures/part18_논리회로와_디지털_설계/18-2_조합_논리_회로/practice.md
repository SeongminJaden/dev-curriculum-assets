# 18-2. 조합 논리 회로 — 실습

> **참고** 이 실습은 제출물이 아니라, 가산기를 진리표에서 직접 도출하고 지연을 손으로 계산해보는 과정입니다.

## 실습 1 — 반가산기·전가산기 직접 구현

- [ ] `xor(a, b)`와 `and_(a, b)`만 사용하는 함수 `half_adder(a, b)`를 작성해 `(carry, sum)` 튜플을 반환하게 하세요.
- [ ] `half_adder`를 두 번 호출하고 `or_`를 한 번 호출하는 방식으로 `full_adder(a, b, cin)`을 작성하세요.
- [ ] `full_adder`를 여덟 가지 입력 조합(000~111) 전부에 대해 실행하고, 출력이 강의 본문의 전가산기 진리표와 한 줄씩 일치하는지 자동 비교하세요.

## 실습 2 — 4비트 리플 캐리 가산기 조립

- [ ] `full_adder`를 4번 호출해 4비트 가산기 `add4(a_bits, b_bits, c0)`를 작성하세요. 앞 단의 carry를 다음 단의 cin으로 넘기는 부분을 반드시 명시적으로 배선하세요.
- [ ] `add4`로 0~15 사이 모든 값 쌍(256가지)을 더해, 파이썬 정수 덧셈 결과의 하위 4비트 및 5번째 비트(자리올림)와 전부 일치하는지 검증하세요.
- [ ] `full_adder` 호출 횟수를 세는 카운터를 추가해, 8비트로 확장하면 호출 횟수가 몇 배로 늘어나는지 숫자로 확인하세요.

## 실습 3 — 자리올림 전파 지연 관찰

- [ ] 각 전가산기가 "cin이 확정된 시각 + 1"에 자기 carry를 확정한다고 가정하고, `add4`가 각 자리의 carry를 확정하는 시각을 리스트로 출력하도록 코드를 수정하세요.
- [ ] $A = 1111_2$, $B = 0001_2$을 더할 때와 $A = 0000_2$, $B = 0000_2$을 더할 때 최종 확정 시각이 어떻게 다른지 비교하세요.
- [ ] 비트 수를 4, 8, 16, 32로 늘려가며 최종 확정 시각을 표로 정리하고, 그것이 $O(n)$ 형태로 늘어나는지 그래프로 그려보세요.

## 실습 4 — 캐리 예측과 뺄셈 확장

- [ ] 각 자리의 $G_i = A_i \cdot B_i$와 $P_i = A_i \oplus B_i$를 계산하고, $C_1 = G_0 + P_0 C_0$, $C_2 = G_1 + P_1 G_0 + P_1 P_0 C_0$을 직접 코드로 구현해 리플 방식의 결과와 값이 같은지 확인하세요.
- [ ] 위 두 식이 참조하는 신호를 나열해, $C_2$가 $C_1$을 전혀 참조하지 않는다는 사실을 눈으로 확인하고 그 의미를 한 문장으로 적으세요.
- [ ] `add4`에 `sub` 플래그를 추가해, `sub=1`이면 각 $B_i$를 XOR로 반전하고 $C_0$에 1을 넣도록 고치세요. 그 뒤 $5 - 3$, $3 - 5$를 계산해 2의 보수 결과가 맞는지 확인하세요.

> **팁** 실습 3에서 지연을 "시각"으로 모델링하는 것이 낯설다면, 각 게이트를 통과할 때마다 카운터를 1씩 올리는 방식이라고 생각하면 됩니다. 실제 하드웨어 시뮬레이터가 하는 일도 본질적으로 이것입니다.
