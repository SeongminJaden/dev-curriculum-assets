# 18-3. 멀티플렉서 · 디코더 · 인코더 — 실습

> **참고** 이 실습은 제출물이 아니라, 선택·해독 회로를 직접 만들어 CPU 내부의 신호 흐름을 손으로 재현해보는 과정입니다.

## 실습 1 — 멀티플렉서 만들기

- [ ] `and_`, `or_`, `not_` 세 함수만 사용해 `mux2(d0, d1, s)`를 구현하고, $Y = \overline{S}D_0 + S D_1$ 식을 코드에 그대로 옮겼는지 확인하세요.
- [ ] `mux2`를 세 번 호출하는 방식으로 `mux4(d0, d1, d2, d3, s1, s0)`을 만들고, 16가지 입력 조합에 대해 올바른 입력이 선택되는지 표로 출력하세요.
- [ ] `mux4`의 네 데이터 입력에 상수 0/1을 꽂아 2변수 XOR 함수를 구현해보고, FPGA의 LUT이 왜 MUX로 만들어지는지 한 문장으로 적으세요.

## 실습 2 — 디코더로 메모리 흉내내기

- [ ] `decode3(a2, a1, a0)`를 작성해 8개 원소 리스트를 반환하되, 정확히 한 자리만 1이 되도록(원-핫) 구현하세요.
- [ ] 길이 8의 리스트를 메모리로 삼고, `decode3`의 출력과 각 워드를 AND한 뒤 전부 OR로 합쳐 "읽기" 동작을 구현하세요. 인덱싱 연산자 `[]`를 쓰지 않고 구현하는 것이 핵심입니다.
- [ ] 주소 0~7을 모두 넣어보고 인덱싱으로 읽은 값과 결과가 같은지 검증한 뒤, 몇 개의 AND·OR 게이트가 필요했는지 세어보세요.

## 실습 3 — 인코더의 결함 재현하기

- [ ] `encode4(d3, d2, d1, d0)`를 $A_1 = D_2 + D_3$, $A_0 = D_1 + D_3$ 식 그대로 구현하세요.
- [ ] $D_1 = D_2 = 1$을 넣었을 때 출력이 무엇인지 확인하고, 그 값이 왜 잘못된 답인지 한 줄로 적으세요.
- [ ] 입력이 전부 0일 때와 $D_0$만 1일 때의 출력을 비교하고, 두 상황을 구분하려면 어떤 신호가 추가로 필요한지 설명을 적으세요.

## 실습 4 — 우선순위 인코더 구현

- [ ] $A_1 = D_3 + D_2$, $A_0 = D_3 + D_1 \cdot \overline{D_2}$, $V = D_3 + D_2 + D_1 + D_0$ 세 식으로 `priority_encode4`를 구현하세요.
- [ ] 16가지 입력 조합 전부를 출력하고, 입력이 여러 개 1인 모든 경우에 항상 가장 큰 번호가 나오는지 확인하세요.
- [ ] $A_0$ 식에서 $\overline{D_2}$ 항을 지운 버전도 만들어, 어떤 입력에서 결과가 달라지는지 정확히 찾아내고 그 항의 역할을 적으세요.
- [ ] 32비트 정수를 받아 최상위 1비트의 위치를 반환하는 함수를 우선순위 인코딩 방식으로 작성하고, 부동소수점 정규화에서 왜 이 회로가 필요한지 조사해 두 문장으로 정리하세요.

> **팁** 실습 2에서 인덱싱을 금지하는 이유는, 하드웨어에는 `[]`가 없기 때문입니다. 메모리 읽기는 언제나 "디코더가 한 줄을 켜고, 켜진 줄의 값만 AND로 통과시켜 OR로 모으는" 물리적 과정입니다. 이것을 코드로 한 번 재현해보면 주소라는 개념이 훨씬 구체적으로 다가옵니다.
