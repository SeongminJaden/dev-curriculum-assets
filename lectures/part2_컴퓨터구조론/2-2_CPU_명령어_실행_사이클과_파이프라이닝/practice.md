# 2-2. 실습 — CPU 명령어 실행 사이클과 파이프라이닝

> **참고** 이 실습은 제출물이 아니라, 파이프라인이 왜 "빨라지는가"를 숫자로 직접 확인해보는 과정입니다.

## 1. 처리량 직접 계산하기

- [ ] 5단계 파이프라인(IF-ID-EX-MEM-WB)에서 명령어 20개를 처리한다고 가정하고, `사이클 수 = k + (n - 1)` 공식에 값을 대입해 총 사이클 수를 계산해보세요.
- [ ] 같은 20개 명령어를 파이프라인 없이 처리한다면(`n × k`) 몇 사이클이 걸리는지 계산해보세요.
- [ ] 두 값을 비교해, 파이프라인 적용 시 속도 향상이 몇 배에 가까워지는지 적어보세요.

## 2. 세탁소 비유를 다른 상황으로 바꿔보기

- [ ] 본문의 "세탁-건조-개기-정리" 비유를 여러분의 일상 속 4단계 반복 작업(예: 요리, 설거지, 서류 처리)으로 바꿔 적어보세요.
- [ ] 그 작업에서 파이프라인을 적용하지 않았을 때와 적용했을 때 걸리는 시간을 표로 비교해보세요.
- [ ] 그 작업에서 발생할 수 있는 "구조적 해저드"(자원이 하나뿐이라 기다려야 하는 상황)를 한 가지 찾아 적어보세요.

## 3. 해저드 구분 연습

다음 상황이 구조적·데이터·제어 해저드 중 무엇에 해당하는지 표시해보세요.

- [ ] 두 명령어가 동시에 같은 메모리 포트를 쓰려고 한다. → 해저드 종류: ______
- [ ] `if` 조건문의 결과가 아직 계산되지 않아 다음에 실행할 명령어를 확정할 수 없다. → 해저드 종류: ______
- [ ] 바로 앞 명령어가 계산한 값을 다음 명령어가 곧바로 사용해야 한다. → 해저드 종류: ______

## 4. 스스로 점검하기

- [ ] "파이프라이닝이 지연시간이 아니라 처리량을 늘린다"는 문장의 의미를 예시를 들어 설명할 수 있다.
- [ ] 포워딩과 분기 예측이 각각 어떤 해저드에 대응하는 기법인지 구분해 말할 수 있다.
- [ ] 파이프라인을 무한히 깊게 만들 수 없는 이유를 두 가지 이상 말할 수 있다.
