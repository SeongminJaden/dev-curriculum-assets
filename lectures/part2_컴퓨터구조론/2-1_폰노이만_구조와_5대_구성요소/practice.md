# 2-1. 실습 — 폰노이만 구조와 5대 구성요소

> **참고** 이 실습은 제출물이 아니라, 강의에서 배운 구조를 스스로의 언어로 설명해보는 과정입니다. 정답을 맞히는 것보다, 왜 그렇게 생각했는지 스스로에게 설명할 수 있는지가 더 중요합니다.

## 1. 5대 구성요소 대응시키기

다음 순서로 정리해보세요.

- [ ] 평소 자주 쓰는 프로그램(예: 웹 브라우저, 메모 앱)을 하나 골라주세요.
- [ ] 그 프로그램을 실행하는 동안 "입력장치 → 기억장치 → 제어장치/연산장치 → 출력장치"로 이어지는 흐름을 한 문장씩 적어보세요.
- [ ] 각 문장에 등장하는 물리적 장치(키보드, SSD, CPU, 모니터 등)를 5대 구성요소 중 무엇에 해당하는지 표로 정리해보세요.

## 2. 병목 지점 추론하기

- [ ] 위에서 고른 프로그램의 흐름 중, CPU와 기억장치 사이의 버스를 가장 많이 사용할 것 같은 지점을 하나 골라보세요.
- [ ] 그 지점을 고른 이유를 2~3문장으로 적어보세요. (힌트: 데이터의 양이 큰가, 반복 접근이 잦은가)
- [ ] 만약 그 지점이 느려진다면, 사용자 입장에서는 어떤 증상으로 체감될지 적어보세요. (예: "화면이 멈춘다", "로딩이 길어진다")

## 3. 저장 프로그램 방식 확인하기

- [ ] "프로그램과 데이터가 같은 기억장치에 저장된다"는 문장이 실제로 무엇을 의미하는지, 자신의 말로 다시 설명해보세요.
- [ ] 만약 프로그램(명령어)과 데이터가 물리적으로 완전히 분리된 기억장치에 저장된다면(하버드 구조), 어떤 점이 더 안전해질지 한 가지를 추측해 적어보세요.

## 4. 스스로 점검하기

아래 질문에 막힘없이 답할 수 있는지 확인해보세요.

- [ ] 제어장치와 연산장치의 역할 차이를 한 문장씩으로 구분해 말할 수 있다.
- [ ] "폰노이만 병목"이 왜 생기는지, 버스라는 단어를 사용해 설명할 수 있다.
- [ ] 파이프라이닝과 메모리 계층이 각각 이 병목의 어떤 부분을 완화하는지 한 문장씩 말할 수 있다.
