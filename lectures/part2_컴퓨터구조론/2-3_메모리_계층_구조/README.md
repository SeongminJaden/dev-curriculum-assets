# Part 2. 컴퓨터 구조론

## 2-3. 메모리 계층 구조 — 캐시는 왜 "더 빠른데 더 작은" 메모리를 여러 겹으로 두는가?

> **학습 목표**
> - 레지스터-L1/L2/L3 캐시-주기억장치-보조기억장치로 이어지는 메모리 계층 구조의 순서와 각 계층의 역할을 설명할 수 있다.
> - 지역성 원리(시간적·공간적 지역성)가 왜 캐시를 효과적으로 만드는지 설명할 수 있다.
> - 적중률·미스율·AMAT 공식을 직접 계산하고, 이 수치가 실제 성능 튜닝에서 왜 쓰이는지 설명할 수 있다.

> **개요** 2-1강에서 CPU와 기억장치 사이의 버스가 병목이 된다고 배웠습니다. 이번 강의는 그 병목을 완화하는 대표적인 해법인 메모리 계층 구조를 다룹니다. "더 빠른 메모리를 그냥 다 크게 만들면 되지 않을까"라는 질문에서 출발해, 왜 계층을 나누는 것이 최선의 절충안인지 살펴봅니다.

---

## 1. 왜 메모리를 한 종류로 통일하지 않을까요?

> **Q.** 레지스터만큼 빠른 메모리를 기억장치 전체 용량만큼 만들면 되지 않을까요?
> **A.** 만들 수는 있지만, 그러면 감당할 수 없이 비싸고 커집니다. 빠른 메모리(SRAM 계열)는 데이터를 저장하는 회로 하나마다 여러 개의 트랜지스터를 써서 즉시 접근 가능하게 만들지만, 그만큼 칩 면적과 비용이 큽니다. 느린 메모리(DRAM, 그리고 SSD/HDD)는 트랜지스터 수가 적어 훨씬 저렴하고 큰 용량을 만들 수 있지만, 접근에 더 오랜 시간이 걸립니다. <u>**"빠르고 크고 싸다"를 동시에 만족하는 메모리는 존재하지 않으며**</u>, 메모리 계층 구조는 이 트레이드오프를 계층별로 나눠 절충한 결과입니다.

## 2. 메모리 계층 구조 — 레지스터부터 보조기억장치까지

> **개요** CPU에서 멀어질수록 느려지고 커지는 6단계 계층을 살펴봅니다.

![메모리 계층 구조 피라미드](https://raw.githubusercontent.com/SeongminJaden/dev-curriculum-assets/main/lectures/part2_컴퓨터구조론/2-3_메모리_계층_구조/assets/diagram1_hierarchy_pyramid.svg)

메모리 계층은 CPU에 가까운 순서대로 다음과 같이 배열됩니다.

1. **레지스터 (Register)** — ALU 바로 옆에 위치한 가장 빠른 저장 공간입니다. 연산에 즉시 쓰일 값만 임시로 담습니다.
2. **L1 캐시** — 코어마다 하나씩 존재하며, 명령어용과 데이터용으로 나뉘어 있는 경우가 많습니다.
3. **L2 캐시** — L1보다 크지만 느리며, 코어 하나 또는 소수의 코어가 공유합니다.
4. **L3 캐시** — 하나의 CPU 안의 모든 코어가 공유하는 마지막 캐시 계층입니다.
5. **주기억장치 (Main Memory, RAM)** — 실행 중인 프로그램과 데이터를 담는 공간으로, 전원이 꺼지면 내용이 사라집니다(휘발성).
6. **보조기억장치 (Secondary Storage, SSD/HDD)** — 전원이 꺼져도 내용이 유지되는(비휘발성) 저장소로, 용량은 가장 크지만 접근 속도는 가장 느립니다.

> **참고** 레지스터부터 L3 캐시까지는 모두 CPU 다이 내부 또는 바로 옆에 위치하며, 대부분 SRAM(Static RAM) 기술을 씁니다. 주기억장치는 DRAM(Dynamic RAM)으로, 구조가 더 단순해 저렴하지만 주기적으로 재충전(refresh)해야 값이 유지됩니다. 보조기억장치는 아예 다른 물리적 원리(플래시 메모리 또는 자기 디스크)를 씁니다.

## 3. 캐시가 통하는 이유 — 지역성 원리

> **개요** 캐시가 작은데도 대부분의 요청을 처리할 수 있는 이유, 즉 프로그램의 메모리 접근 패턴에 존재하는 규칙성을 살펴봅니다.

캐시는 기억장치 전체가 아니라 극히 일부만 담을 수 있는 작은 공간입니다. 그런데도 대부분의 CPU 요청이 캐시에서 해결되는 이유는 프로그램의 메모리 접근이 무작위가 아니라 <u>**지역성(Locality)**</u>이라는 규칙적인 패턴을 따르기 때문입니다.

- **시간적 지역성 (Temporal Locality)** — 한 번 접근한 데이터는 가까운 미래에 다시 접근될 가능성이 높습니다. 반복문 안의 변수, 함수 호출 중 반복적으로 참조하는 값이 대표적입니다.
- **공간적 지역성 (Spatial Locality)** — 어떤 주소를 접근하면, 그 주소와 가까운 주소도 곧 접근될 가능성이 높습니다. 배열을 순서대로 순회하는 코드, 그리고 명령어 자체가 기억장치에 순서대로 저장되어 순차적으로 인출되는 것이 대표적입니다.

> **Q.** 캐시는 지역성을 어떻게 실제로 활용할까요?
> **A.** 캐시는 데이터를 요청받은 한 개의 값만 가져오지 않고, 그 값이 포함된 일정 크기의 블록(캐시 라인, 보통 수십 바이트 단위)을 통째로 가져옵니다. 공간적 지역성 덕분에, 방금 가져온 블록 안의 다른 값들도 곧 쓰일 확률이 높기 때문입니다. 또한 한 번 캐시에 올라온 데이터는 최근에 쓰인 데이터가 우선적으로 유지되도록 관리되는데(LRU와 같은 교체 정책), 이는 시간적 지역성을 활용하는 방식입니다.

### 실생활 예시 — 책상, 책장, 도서관

> **해석** 아래 표는 메모리 계층을 "필요한 자료를 찾는 장소"에 빗댄 것입니다. 자료를 찾을 때 가장 가까운 곳부터 확인하고, 없으면 점점 먼 곳으로 이동하는 순서가 캐시의 탐색 순서와 정확히 일치합니다.

| 메모리 계층 | 실생활 대응 | 접근 속도 체감 |
|---|---|---|
| 레지스터 | 지금 손에 쥐고 있는 펜과 종이 | 즉시 |
| L1/L2 캐시 | 책상 위에 펼쳐 놓은 자료 | 손을 뻗으면 바로 |
| L3 캐시 | 같은 방 책장에 꽂힌 책 | 일어나서 몇 걸음 |
| 주기억장치 | 같은 건물 다른 층 서고 | 엘리베이터를 타고 이동 |
| 보조기억장치 | 다른 지역의 도서관에 상호대차 신청 | 며칠을 기다려야 함 |

> **팁** 자주 쓰는 자료를 책상 위(캐시)에 그대로 펼쳐 두는 것이 매번 서고(주기억장치)까지 걸어가는 것보다 압도적으로 효율적인 것처럼, 소프트웨어에서도 배열을 순차적으로 접근하는 코드가 무작위로 접근하는 코드보다 훨씬 빠릅니다. 이것이 실무에서 "캐시 친화적인 코드"를 작성해야 한다고 말하는 근거입니다.

## 4. 적중률과 평균 접근시간 — 캐시 성능을 수치로 표현하기

> **개요** 캐시가 실제로 얼마나 효과적인지 표현하는 표준 공식인 적중률과 AMAT(평균 메모리 접근시간)를 다룹니다.

![캐시 접근 흐름과 AMAT 공식](https://raw.githubusercontent.com/SeongminJaden/dev-curriculum-assets/main/lectures/part2_컴퓨터구조론/2-3_메모리_계층_구조/assets/diagram2_amat_flow.svg)

CPU가 데이터를 요청하면 가장 가까운(가장 빠른) 계층부터 확인하고, 없으면 한 단계씩 더 먼 계층으로 내려갑니다. 이 과정을 수치화하는 핵심 지표는 다음과 같습니다.

- **적중 (Hit)** — 요청한 데이터가 해당 계층에 있는 경우
- **미스 (Miss)** — 요청한 데이터가 해당 계층에 없어, 더 아래 계층까지 내려가야 하는 경우
- **적중률 (Hit Rate)** = 적중 횟수 ÷ 전체 접근 횟수
- **미스율 (Miss Rate)** = 1 − 적중률 = 미스 횟수 ÷ 전체 접근 횟수

이 지표를 바탕으로, 캐시를 포함한 전체 메모리 시스템의 체감 성능을 나타내는 표준 공식이 <u>**평균 메모리 접근시간(Average Memory Access Time, AMAT)**</u>입니다.

```
AMAT = 적중시간(Hit Time) + 미스율(Miss Rate) × 미스 페널티(Miss Penalty)
```

- **적중시간(Hit Time)**: 해당 계층에서 데이터를 바로 찾았을 때 걸리는 시간
- **미스 페널티(Miss Penalty)**: 미스가 발생해 다음 계층까지 내려가 데이터를 가져오는 데 추가로 걸리는 시간

> **Q.** 이 공식이 왜 "적중시간 + 미스율 × 미스 페널티"라는 형태일까요?
> **A.** 모든 접근은 일단 해당 계층에서 확인하는 시간(적중시간)이 반드시 듭니다. 그리고 그중 미스율만큼의 비율로는 추가로 미스 페널티라는 시간을 더 써야 합니다. 즉 "기본으로 드는 시간 + 실패했을 때만 추가로 드는 시간의 기댓값"을 더한 것이 평균 접근시간입니다. 계층이 여러 겹이라면 이 공식을 재귀적으로 적용합니다. 예를 들어 L1의 미스 페널티는 곧 "L2의 AMAT"가 됩니다.

> **참고** 예를 들어 L1 캐시의 적중시간이 1(단위 사이클), 적중률이 95%(미스율 5%)이고, 미스 페널티(L2까지 내려가는 데 드는 시간)가 10이라면, AMAT = 1 + 0.05 × 10 = 1.5가 됩니다. 적중률이 90%로 떨어지면 AMAT = 1 + 0.10 × 10 = 2.0으로, 체감 성능이 33% 이상 나빠집니다. 적중률의 작은 하락이 전체 성능에 큰 영향을 미치는 이유가 여기에 있습니다.

> **팁** 실무에서 서버의 CPU 사용률은 낮은데 응답 속도가 느린 경우, 캐시 미스율이 높은 것이 원인일 때가 많습니다. 프로파일링 도구로 캐시 적중률을 측정하고, 자주 접근하는 데이터의 크기를 캐시 용량에 맞추거나 데이터 접근 순서를 지역성에 맞게 재배열하는 것이 실질적인 성능 튜닝 기법입니다. 데이터베이스의 버퍼 캐시, 애플리케이션의 인메모리 캐시(Redis 등)도 이 원리를 응용한 것입니다.

> **핵심**
> - 메모리 계층은 레지스터 → L1 → L2 → L3 캐시 → 주기억장치 → 보조기억장치 순으로 CPU에서 멀어질수록 <u>**느려지고 커지고 저렴해집니다.**</u>
> - 캐시가 작은데도 효과적인 이유는 프로그램의 메모리 접근이 <u>**시간적·공간적 지역성**</u>을 따르기 때문입니다.
> - **AMAT = 적중시간 + 미스율 × 미스 페널티** 공식은 캐시 성능을 정량화하는 표준 지표이며, 적중률의 작은 변화가 전체 성능에 큰 영향을 미칩니다.

## 요약

> **핵심**
> - 메모리 계층 구조는 "빠르고 크고 싸다"를 동시에 만족하는 메모리가 없다는 물리적 제약의 절충안이며, **레지스터-L1/L2/L3 캐시-주기억장치-보조기억장치**의 순서로 구성됩니다.
> - 이 구조가 효과적으로 작동하는 근거는 프로그램의 메모리 접근이 따르는 **시간적·공간적 지역성**입니다.
> - **적중률·미스율·AMAT** 공식은 캐시 성능을 정량적으로 표현하는 표준 도구이며, 실제 서버 성능 튜닝과 데이터베이스·애플리케이션 캐시 설계에 그대로 쓰입니다.
