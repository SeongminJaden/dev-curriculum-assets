# 10-3 실습 — 모니터링과 관측성(Observability)

> **참고** 제출물이 아니라, 오늘 배운 세 기둥 개념을 실제 장애 상황에 적용해보는 실습입니다.

## 실습 1. 세 기둥 구분해보기

- [ ] 최근에 겪었거나 들어본 서비스 장애 사례를 하나 떠올려보세요. (없다면 가상의 시나리오를 만들어도 됩니다.)
- [ ] 그 장애를 파악하는 과정에서 로그, 메트릭, 트레이스 중 어떤 데이터가 각각 어떤 질문에 답했을지(또는 답했어야 했을지) 정리해보세요.

## 실습 2. 모니터링 규칙 설계하기

- [ ] 여러분이 운영하는 웹 서비스가 있다고 가정해보세요.
- [ ] 어떤 지표를 모니터링 대상으로 삼아야 할지 3가지를 정해보세요. (예: 평균 응답 시간, 에러율 등)
- [ ] 각 지표가 "정상 범위를 벗어났다"고 판단할 기준을 함께 적어보세요.

## 실습 3. 분산 트레이스 그려보기

- [ ] 사용자가 겪을 만한 서비스 시나리오 하나(예: 티켓 예매, 배달 주문)를 정해보세요.
- [ ] 이 요청이 거칠 것으로 예상되는 서비스를 3~4개 나열해보세요.
- [ ] 각 서비스에 걸리는 시간을 가정하고, 트레이스 ID로 묶었을 때 어느 구간이 병목인지 표시해보세요.

## 생각해볼 질문

- [ ] 메트릭 대시보드에서 "에러율이 급증했다"는 사실만 확인했을 때, 왜 그것만으로는 원인을 확정할 수 없을까요?
