# 10-2 실습 — 인프라 as 코드(IaC)와 배포 전략

> **참고** 제출물이 아니라, 오늘 배운 IaC와 배포 전략 개념을 직접 적용해보는 실습입니다.

## 실습 1. 설정 드리프트 시나리오 만들어보기

- [ ] 서버 3대를 사람이 손으로 설정한다고 가정해보세요.
- [ ] 시간이 지나며 이 서버들 사이에 어떤 차이가 생길 수 있을지 구체적인 예시를 3가지 적어보세요.
- [ ] 그중 하나가 실제 장애로 이어진다면 어떤 상황일지 한 문단으로 적어보세요.

## 실습 2. 배포 전략 선택하기

- [ ] 아래 두 상황 각각에 블루/그린, 카나리, 롤링 배포 중 어느 것이 더 적합할지 정하고 이유를 적어보세요.
  - [ ] 하루 방문자가 적은 사내 관리자 페이지의 UI 개편
  - [ ] 전 세계 수천만 사용자가 쓰는 소셜미디어 앱의 추천 알고리즘 교체
- [ ] 선택한 전략에서 문제가 발견되었을 때 롤백이 구체적으로 어떻게 이루어질지 설명해보세요.

## 실습 3. 선언적 사고로 바꿔보기

- [ ] "서버에 A 프로그램을 설치하고, B 설정 파일을 복사하고, C 서비스를 재시작한다"는 명령형 절차를 하나 떠올려보세요.
- [ ] 이를 "최종적으로 서버가 어떤 상태여야 하는가"를 설명하는 선언적 문장으로 바꿔 적어보세요.

## 생각해볼 질문
