# 9-3. 시스템·네트워크 보안 기초 — 실습

> **참고** 이 실습은 제출물이 아니라, 킬체인 모델과 방어 장비의 대응 관계를 스스로 정리해보는 과정입니다. 실제 네트워크나 타인의 시스템을 대상으로 스캔·침투를 시도하지 마세요.

## 실습 1 — 킬체인 단계 나눠보기

- [ ] 뉴스나 보안 블로그에서 공개된 침해사고 사례(회사명이 언급된 실제 사건도 좋습니다) 하나를 찾아보세요.
- [ ] 그 사고의 진행 과정을 킬체인 7단계(정찰, 무기화, 전달, 익스플로잇, 설치, C2, 목표 달성) 중 어디에 해당하는지 나눠 정리해보세요.
- [ ] 각 단계에서 만약 어떤 방어 장비가 있었다면 공격 흐름을 끊을 수 있었을지 적어보세요.

## 실습 2 — 방화벽 규칙 설계해보기

- [ ] 가상의 소규모 회사 사내망을 하나 상상해보세요. (웹 서버 1대, 내부 직원 PC 여러 대, 데이터베이스 서버 1대)
- [ ] 이 사내망에 필요한 방화벽 규칙을 5개 이상 나열해보세요. (예: 외부에서 웹 서버의 443 포트만 허용, 데이터베이스 서버는 외부 접근 전면 차단 등)
- [ ] 작성한 규칙만으로 막을 수 없는 공격 경로가 있는지 생각해보고, 왜 막을 수 없는지 설명해보세요.

## 실습 3 — IDS/IPS 탐지 방식 비교하기

- [ ] 시그니처 기반 탐지와 이상 탐지 각각의 장단점을 표로 정리해보세요.
- [ ] "이미 알려진 공격 패턴을 재사용하는 공격"과 "완전히 새로운 방식의 공격" 각각에 어떤 탐지 방식이 더 유리할지 판단해보세요.
- [ ] IDS와 IPS 중 어떤 상황에서 IPS의 자동 차단이 오히려 위험할 수 있는지(정상 트래픽 오탐 시 서비스 장애) 생각해보세요.

## 실습 4 — 심층 방어 설계도 그려보기

- [ ] 강의의 diagram2(네트워크 경계 다이어그램)를 참고해, 자신이 상상한 조직의 네트워크 구조를 손으로 그려보세요.
- [ ] 인터넷, 방화벽, DMZ, IDS/IPS, 내부망을 순서대로 배치하고, 각 경계에 어떤 규칙 또는 감시가 필요한지 적어보세요.
- [ ] 만약 방화벽 하나가 뚫렸다고 가정했을 때, 다음 방어선이 무엇을 막아줄 수 있는지 적어보세요.

## 실습 5 — 핵심 질문 다시 답해보기

- [ ] "방화벽과 침입 탐지는 각각 어떤 공격 단계를 막기 위함인가"라는 질문에 책을 보지 않고 세 문장 이내로 답해보세요.
- [ ] 작성한 답을 강의 2절, 3절의 Q&A와 비교해 빠진 논리가 있는지 확인해보세요.
