# 9-1. 웹 애플리케이션 보안 기초 — 실습

> **참고** 이 실습은 제출물이 아니라, 강의에서 배운 "신뢰 경계"라는 사고방식을 직접 코드와 서비스에 적용해보는 과정입니다. 실제 공격 스크립트를 작성하거나 실행하지 말고, 개념적으로 분석하는 데 집중하세요.

## 실습 1 — 신뢰 경계 찾아보기

- [ ] 최근에 만들었거나 사용해본 웹 서비스의 로그인 폼 또는 검색 기능 하나를 골라보세요.
- [ ] 사용자가 입력한 값이 서버 내부에서 어떤 문자열(쿼리, HTML, 파일 경로 등)에 합쳐지는지 상상해보고, 신뢰 경계가 어디에 그어져야 하는지 표시해보세요.
- [ ] 그 경계에서 입력값이 이스케이프 없이 그대로 사용된다면 어떤 문제가 생길 수 있는지 한두 문장으로 적어보세요.

## 실습 2 — 준비된 문(Prepared Statement)으로 바꿔보기

- [ ] 사용하는 언어(또는 익숙한 언어)에서 문자열을 이어 붙여 SQL 쿼리를 만드는 예시 코드를 하나 작성하세요. (실제 데이터베이스에 연결하지 않아도 됩니다.)
- [ ] 같은 쿼리를 준비된 문(Prepared Statement) 또는 ORM의 파라미터 바인딩 방식으로 다시 작성해보세요.
- [ ] 두 코드의 차이가 "입력값이 쿼리 구조에 영향을 줄 수 있는가"라는 질문에 각각 어떻게 답하는지 비교해보세요.

## 실습 3 — XSS 방어 지점 분석하기

- [ ] 게시판이나 댓글 기능이 있는 서비스를 하나 떠올려보세요.
- [ ] 사용자가 입력한 텍스트가 화면의 어느 위치(본문, 속성값, 링크 등)에 표시되는지 나열해보세요.
- [ ] 각 위치마다 이스케이프 처리가 되어 있는지, 되어 있지 않다면 어떤 위험이 있을지 표로 정리해보세요.

## 실습 4 — 인증과 인가 구분하기

- [ ] 자신이 사용하는 서비스에서 "로그인했는가"와 "이 자원에 접근할 권한이 있는가"를 서버가 각각 어느 시점에 확인할지 추측해보세요.
- [ ] 만약 URL의 게시글 번호를 다른 사용자의 번호로 바꿔도 접근이 되는지(IDOR 가능성) 개념적으로만 생각해보고, 왜 그런 결과가 나올지 서버 로직 관점에서 설명해보세요. (실제로 타인의 계정이나 서비스에 시도하지 마세요.)

## 실습 5 — OWASP Top 10 최신판 확인하기

- [ ] OWASP 공식 사이트에서 최신 Top 10 카테고리 10개를 찾아 목록을 적어보세요.
- [ ] 그중 이 강의에서 다룬 인젝션, 접근 제어와 관련된 카테고리가 각각 몇 번째인지 확인해보세요.
- [ ] 순위가 이전 개정판과 비교해 어떻게 바뀌었는지, 그 변화가 무엇을 시사하는지 한 문단으로 정리해보세요.
