# 9-2. 암호화와 인증 프로토콜 — 실습

> **참고** 이 실습은 제출물이 아니라, 대칭키·비대칭키의 차이와 TLS 핸드셰이크의 흐름을 스스로 손으로 따라가 보는 과정입니다.

## 실습 1 — 대칭키와 비대칭키 구분하기

- [ ] 평소 사용하는 서비스에서 암호화가 쓰이는 상황을 세 가지 찾아보세요. (예: 파일 압축 프로그램의 비밀번호, 메신저의 종단간 암호화, HTTPS 접속)
- [ ] 각 상황이 대칭키에 가까운지 비대칭키에 가까운지 판단하고, 그렇게 판단한 근거를 적어보세요.
- [ ] "속도가 중요한가" 또는 "사전에 키를 공유할 방법이 없는가"라는 두 가지 질문으로 각 상황을 다시 분류해보세요.

## 실습 2 — RSA의 수학적 구조 손으로 따라가기

- [ ] 아주 작은 소수 두 개(예: `p = 5`, `q = 11`)를 골라 `n = p × q`를 계산해보세요.
- [ ] `φ(n) = (p-1)(q-1)`을 계산해보세요.
- [ ] `φ(n)`과 서로소인 작은 정수 `e`를 하나 골라보세요. (예: `e = 3`)
- [ ] `e·d ≡ 1 (mod φ(n))`을 만족하는 `d`를 손으로 찾아보세요.
- [ ] 이 실습에서 사용한 소수가 왜 실제 서비스에서는 훨씬 커야 하는지(2048비트 이상), 소인수분해의 계산 난이도와 연결지어 설명해보세요.

## 실습 3 — TLS 핸드셰이크 순서 재구성하기

- [ ] 강의 다이어그램을 가리고, ClientHello부터 세션 키 합의까지의 단계를 자신의 언어로 순서대로 적어보세요.
- [ ] 각 단계에서 "이 단계가 없다면 어떤 공격이 가능해지는가"를 하나씩 적어보세요. (예: 인증서 검증이 없다면 어떤 문제가 생기는가)
- [ ] 다시 다이어그램과 비교해 빠뜨린 단계가 있는지 확인해보세요.

## 실습 4 — 브라우저에서 인증서 직접 확인하기

- [ ] 평소 사용하는 웹사이트에 접속해 주소창의 자물쇠 아이콘을 클릭하고 인증서 정보를 열어보세요.
- [ ] 발급기관(CA), 유효기간, 적용 도메인을 확인하고 기록해보세요.
- [ ] 같은 방식으로 다른 사이트 하나를 더 확인해, 두 인증서의 발급기관이 같은지 다른지 비교해보세요.

## 실습 5 — 핵심 질문 다시 답해보기

- [ ] "비대칭 암호화 없이 TLS 핸드셰이크가 불가능한 이유"를 책을 보지 않고 세 문장 이내로 적어보세요.
- [ ] 작성한 답을 강의 5절의 Q&A와 비교해, 빠진 논리가 있는지 확인해보세요.
