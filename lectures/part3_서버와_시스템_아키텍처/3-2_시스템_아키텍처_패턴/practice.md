# 3-2. 시스템 아키텍처 패턴 — 실습

> **참고** 실습은 제출물이 아니라, 오늘 배운 세 가지 아키텍처 패턴을 실제 서비스에 대입해보는 과정입니다.

- [ ] 자주 사용하는 앱이나 웹 서비스를 하나 골라, 그 서비스가 가지고 있을 법한 기능을 5개 이상 나열해보세요. (예: 주문, 결제, 리뷰, 알림, 검색)
- [ ] 나열한 기능들을 "같은 팀이 관리할 것 같은 것끼리" 묶어보고, 이 묶음이 마이크로서비스로 쪼갠다면 각각의 경계가 될 수 있는지 판단해보세요.
- [ ] 그중 트래픽이 특히 몰릴 것 같은 기능 하나를 고르고, 모놀리식이라면 이 기능의 트래픽 급증에 어떻게 대응할지, 마이크로서비스라면 어떻게 다르게 대응할지 비교해 적어보세요.
- [ ] "주문 완료" 이벤트가 발생했을 때, 동기 호출로 처리하면 좋은 후속 작업과 이벤트 기반(비동기)으로 처리해도 괜찮은 후속 작업을 각각 1개 이상 구분해보세요.
