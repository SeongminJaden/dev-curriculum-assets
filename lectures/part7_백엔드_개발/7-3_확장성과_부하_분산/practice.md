# 7-3. 확장성과 부하 분산 — 실습

> **참고** 이 실습은 제출물이 아니라, 오늘 배운 확장성 개념을 실제 서비스 상황에 적용해보는 과정입니다.

## 실습 체크리스트

- [ ] 최근 방문했던 쇼핑몰이나 예매 사이트 중 "대규모 할인 행사" 또는 "선착순 예매"처럼 트래픽이 순간적으로 몰렸던 경험을 떠올리고, 그때 페이지가 느려지거나 오류가 났던 구간이 애플리케이션 서버 때문이었을지, DB 때문이었을지 추측해 이유와 함께 적어보세요.
- [ ] 본인이 직접 서비스를 운영한다고 가정하고, 수직 확장과 수평 확장 중 어느 쪽을 먼저 적용할지 선택한 뒤, 그 선택의 한계(물리적 사양 한도 또는 상태 동기화 문제)가 언제 드러날지 설명해보세요.
- [ ] 로드밸런싱 알고리즘 세 가지(라운드 로빈, 최소 연결, IP 해시) 중, 로그인 세션을 서버 메모리에만 저장하는 서비스에는 어떤 알고리즘이 필요한지 고르고 이유를 적어보세요.
- [ ] 자주 이용하는 서비스에서 "캐싱으로 빨라질 만한 데이터"(자주 조회되지만 잘 바뀌지 않는 정보)와 "캐싱으로는 해결되지 않는 데이터"(매번 새로 쓰여야 하는 정보)를 각각 2가지 이상 찾아보세요.
- [ ] 사용자 수천만 명 규모의 서비스에서 `user_id`를 기준으로 샤딩한다고 가정할 때, 특정 샤드에만 트래픽이 몰리는 핫스팟이 생길 수 있는 샤드 키 설계의 예를 하나 만들어보고, 왜 문제가 되는지 설명해보세요.
