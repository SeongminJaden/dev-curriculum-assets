# 7-2. 인증과 인가 — 실습

> **참고** 이 실습은 제출물이 아니라, 오늘 배운 세션과 JWT의 차이를 스스로 확인해보는 과정입니다.

## 실습 체크리스트

- [ ] 자주 쓰는 웹사이트에 로그인한 뒤, 브라우저 개발자 도구(애플리케이션 탭)에서 쿠키 목록을 확인하고 세션 ID로 보이는 값을 찾아보세요.
- [ ] 어떤 서비스에서 "네이버로 로그인" 또는 "구글로 로그인"을 사용해본 경험을 떠올리고, 이때 비밀번호를 어느 화면에 입력했는지(원래 서비스 화면인지, 네이버·구글 화면인지) 확인해 OAuth 2.0의 위임 구조와 연결해보세요.
- [ ] 세션 기반 인증과 JWT 기반 인증 중, 여러분이 만든다면 "은행 거래 서비스"에는 어느 쪽이 더 적합할지, 즉시 로그아웃(폐기)의 중요성을 근거로 선택하고 이유를 적어보세요.
- [ ] JWT의 페이로드 부분(가운데 부분)만 떼어 Base64URL 디코딩하면 평문으로 읽힌다는 점을 근거로, JWT 페이로드에 절대 넣으면 안 되는 정보의 예시를 3가지 이상 적어보세요.
