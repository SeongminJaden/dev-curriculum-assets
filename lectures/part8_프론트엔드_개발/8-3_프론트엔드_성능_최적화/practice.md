# 8-3. 프론트엔드 성능 최적화 — 실습

> **참고** 실습은 제출물이 아니라, 브라우저 개발자 도구로 실제 페이지의 로딩 과정을 관찰해보는 과정입니다.

- [ ] 자주 쓰는 웹 서비스의 목록형 페이지(상품 목록, 피드 등)를 하나 골라, 개발자 도구의 Network 탭에서 이미지 요청이 한꺼번에 발생하는지, 스크롤에 따라 나눠 발생하는지 확인해보세요.
- [ ] 같은 페이지의 Network 탭에서 자바스크립트 파일이 몇 개로 나뉘어 로드되는지 확인하고, 파일 이름에서 라우트별로 분리된 흔적(코드 스플리팅)을 찾아보세요.
- [ ] 개발자 도구의 Lighthouse(또는 Performance Insights) 탭에서 LCP, CLS 값을 측정해보고, 어떤 요소가 LCP 대상이었는지 확인해보세요.
- [ ] 측정한 결과 중 개선 여지가 있어 보이는 지표 하나를 골라, 이 강의에서 배운 기법(코드 스플리팅, 지연 로딩, 리렌더링 최소화) 중 무엇을 적용하면 좋을지 한 문단으로 정리해보세요.
