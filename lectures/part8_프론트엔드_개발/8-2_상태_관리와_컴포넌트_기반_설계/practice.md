# 8-2. 상태 관리와 컴포넌트 기반 설계 — 실습

> **참고** 실습은 제출물이 아니라, 익숙한 화면 하나를 오늘 배운 기준으로 직접 분해해보는 과정입니다.

- [ ] 자주 쓰는 웹 서비스의 화면 하나를 골라, 눈에 보이는 영역을 3~5개의 컴포넌트로 나눠 이름을 붙여보세요. (예: Header, ProductList, ProductCard, Footer)
- [ ] 나눈 컴포넌트들을 부모-자식 관계에 따라 트리 형태로 그려보세요.
- [ ] 그 화면에서 사용자의 조작(클릭, 입력 등) 하나를 골라, 그 데이터가 어떤 컴포넌트에서 시작해 어떤 콜백을 거쳐 어느 컴포넌트의 상태를 바꾸는지 순서대로 적어보세요.
- [ ] 트리에 그린 상태들을 각각 로컬 상태와 전역 상태로 분류하고, 그렇게 분류한 판단 기준을 한 문장씩 적어보세요.
