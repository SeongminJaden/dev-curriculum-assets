# 4-3. 실습 — 통합 메모리 아키텍처(UMA)

> **참고** 이 실습은 제출물이 아니라, UMA 개념을 실제 제품 스펙에서 확인해보는 과정입니다.

## 1. 제품 스펙에서 UMA 확인하기

- [ ] 애플 공식 웹사이트나 NVIDIA Jetson 제품 페이지에서 "통합 메모리" 또는 "Unified Memory"라는 표현을 찾아, 그 기기가 CPU와 GPU에 각각 몇 GB의 메모리를 어떻게 나눠 쓰는지(혹은 나누지 않는지) 확인해보세요.
- [ ] 반대로 데스크톱용 분리형 그래픽카드 제품 페이지에서 VRAM 용량이 시스템 RAM 용량과 별개로 표기되어 있는지 확인해보세요.

## 2. 복사 비용 추론하기

- [ ] 카메라로 촬영한 4K 해상도 이미지 한 장(대략 24MB)을 분리형 GPU 구조에서 처리한다고 가정할 때, 어느 구간에서 데이터 복사가 발생하는지 이번 강의의 다이어그램을 참고해 단계별로 적어보세요.
- [ ] 같은 작업을 UMA 구조에서 처리한다면 그 단계가 어떻게 바뀌는지 비교해서 적어보세요.

## 3. 설계 선택 시뮬레이션

- [ ] 만약 자신이 배송 로봇의 두뇌를 설계한다면 분리형 GPU와 UMA 기반 SoC 중 무엇을 선택할지, 그리고 그 이유를 이번 강의에서 배운 전력·실시간성·무게 제약을 근거로 두세 문장으로 정리해보세요.
- [ ] 반대로 데이터센터에서 대규모 모델을 학습하는 서버를 설계한다면 어떤 선택이 더 합리적일지도 함께 적어보세요.

## 4. 스스로 점검하기

아래 질문에 막힘없이 답할 수 있는지 확인해보세요.

- [ ] 분리형 GPU 구조에서 데이터 복사가 정확히 어느 두 메모리 사이에서, 어떤 버스를 통해 발생하는지 설명할 수 있다.
- [ ] UMA가 복사를 없애는 원리를 "포인터 전달"이라는 단어를 사용해 설명할 수 있다.
- [ ] UMA의 반대급부(대역폭 공유로 인한 최대 처리량 한계)를 한 문장으로 설명할 수 있다.
- [ ] 로봇·드론이 UMA를 선호하는 이유를 전력, 실시간성, 무게라는 세 가지 키워드로 설명할 수 있다.
