# 16-4. DFS와 BFS — 실습

> **참고** 이 실습은 제출물이 아니라, 같은 그래프를 두 순서로 훑어보며 방문 순서가 만드는 성질 차이를 직접 확인하는 과정입니다.

## 실습 1 — 한 줄만 바꿔 DFS와 BFS 만들기

- [ ] 정점 8개, 간선 10개짜리 무방향 그래프를 인접 리스트로 직접 정의하세요.
- [ ] 강의의 `traverse` 함수처럼 후보 통에서 꺼내는 방식만 바꿔 DFS와 BFS를 각각 실행하고, 두 방문 순서를 나란히 출력해 비교하세요.
- [ ] 같은 DFS를 (a) 재귀와 (b) 명시적 스택 두 가지로 구현하고, 두 방문 순서가 다를 수 있는 이유를 인접 리스트 순회 방향과 연결해 설명하세요.
- [ ] 정점 200,000개가 일직선으로 이어진 사슬 그래프를 만들어 재귀 DFS를 실행하고, 어떤 오류가 나는지 기록하세요.

## 실습 2 — 방문 표시 시점 실험

- [ ] BFS에서 방문 표시를 "큐에 넣을 때" 하는 버전과 "큐에서 꺼낼 때" 하는 버전을 각각 구현하세요.
- [ ] 정점 300개짜리 완전 그래프(모든 정점이 서로 연결)에 두 버전을 돌려 큐에 삽입된 총 횟수를 세고 비교하세요.
- [ ] 방문 표시를 아예 제거한 버전을 사이클이 있는 그래프에 돌려 무한 루프가 발생하는 것을 확인하세요(반복 횟수 제한을 걸어 안전하게).

## 실습 3 — BFS로 최단 경로 구하고 경로 복원하기

- [ ] `.`은 통로, `#`은 벽인 20×20 미로를 문자열 배열로 만들고, 시작점에서 도착점까지 최소 이동 횟수를 BFS로 구하세요.
- [ ] `parent` 배열을 함께 기록해 실제 최단 경로를 좌표 목록으로 복원해 출력하세요.
- [ ] 같은 미로에 DFS를 적용해 찾은 경로 길이가 BFS 결과보다 길어질 수 있음을 확인하고, 그 이유를 한 문장으로 적으세요.
- [ ] 일부 칸의 이동 비용을 2로 바꾼 뒤 BFS를 그대로 돌려 답이 틀리는 것을 확인하세요.

## 실습 4 — 위상 정렬과 순환 의존성 탐지

- [ ] 과목 8개와 선수 과목 관계 10개로 방향 그래프를 만들고, 칸 알고리즘으로 수강 순서를 출력하세요.
- [ ] 결과 리스트의 길이가 정점 수와 같은지 검사하는 코드를 넣고, 일부러 순환 의존성(A→B→C→A)을 추가해 사이클이 탐지되는지 확인하세요.
- [ ] 같은 그래프를 DFS 후위 처리 방식으로도 위상 정렬해, 두 결과가 서로 다르지만 둘 다 유효한 순서임을 간선 방향을 하나씩 검사해 확인하세요.
- [ ] 흰색·회색·검은색 3색 표시를 사용해 방향 그래프의 사이클을 탐지하는 DFS를 구현하고, "이미 방문함" 2색만으로는 왜 부족한지 반례 그래프를 하나 만들어 보이세요.

> **팁** 실습 4의 3색 DFS가 이 강의에서 가장 중요한 대목입니다. "검은색 정점으로 가는 간선은 사이클이 아니다"라는 점을 반례로 직접 확인하면, DFS가 왜 방향 그래프 사이클 탐지에 적합한지 몸으로 이해하게 됩니다.
