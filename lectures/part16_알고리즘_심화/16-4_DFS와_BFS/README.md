# Part 16. 알고리즘 심화

## 16-4. DFS와 BFS — 같은 그래프를 왜 두 가지 순서로 훑는가

> **학습 목표**
> - DFS와 BFS의 동작 원리를 스택·큐라는 자료구조 차이로 설명하고, 각각을 재귀와 반복으로 구현할 수 있다.
> - 방문 표시가 왜 필수인지, 어느 시점에 표시해야 하는지를 근거와 함께 설명하고 복잡도 $O(V+E)$를 도출할 수 있다.
> - BFS가 무가중치 그래프에서 최단 경로를 보장하는 이유를 증명 수준으로 설명하고, 연결 요소·사이클 탐지·위상 정렬에 두 탐색을 적용할 수 있다.

> **개요** 그래프 위의 거의 모든 알고리즘은 결국 "정점들을 어떤 순서로 방문할 것인가"의 문제로 환원됩니다. DFS와 BFS는 그 순서를 정하는 두 가지 근본 전략이고, 놀랍게도 둘의 코드 차이는 **스택을 쓰느냐 큐를 쓰느냐** 한 줄뿐입니다. 그 한 줄이 왜 "최단 경로를 준다/못 준다", "사이클을 찾을 수 있다/없다" 같은 근본적인 성질 차이로 이어지는지 살펴봅니다.

## 1. 두 탐색의 차이는 자료구조 한 줄이다

> **Q.** DFS와 BFS는 완전히 다른 알고리즘처럼 배우는데, 실제로 코드가 그렇게 다른가요?
> **A.** 거의 같습니다. "다음에 방문할 정점 후보를 담아 두는 통"에서 무엇을 먼저 꺼내느냐만 다릅니다. 가장 나중에 넣은 것을 먼저 꺼내면(<u>**스택(stack)**</u>, LIFO) 한 갈래로 깊이 파고들어 DFS가 되고, 가장 먼저 넣은 것을 먼저 꺼내면(<u>**큐(queue)**</u>, FIFO) 가까운 것부터 층층이 퍼져 BFS가 됩니다.

```python
def traverse(graph, start, use_stack):
    visited = {start}
    box = [start]
    order = []
    while box:
        v = box.pop() if use_stack else box.pop(0)   # 이 한 줄이 DFS/BFS를 가름
        order.append(v)
        for nxt in graph[v]:
            if nxt not in visited:
                visited.add(nxt)
                box.append(nxt)
    return order
```

![DFS와 BFS의 방문 순서 비교](https://raw.githubusercontent.com/SeongminJaden/dev-curriculum-assets/main/lectures/part16_알고리즘_심화/16-4_DFS와_BFS/assets/diagram1_dfs_bfs_order.svg)

> **해석** 같은 그래프에서 같은 시작점으로 출발했는데 방문 순서가 전혀 다릅니다. DFS는 갈 수 있는 데까지 내려간 뒤 막히면 되돌아오므로 순서가 세로로 이어지고, BFS는 거리 1인 정점을 모두 방문한 뒤 거리 2로 넘어가므로 순서가 가로로 층을 이룹니다. 이 "층"이 BFS 최단 경로 성질의 근원입니다.

> **참고** BFS는 1945년 콘라트 추제(Konrad Zuse)가 박사 논문에서 처음 고안했지만 출판되지 않았고, 1959년 에드워드 무어(Edward F. Moore)가 미로의 최단 경로를 찾는 맥락에서 독립적으로 재발견해 널리 알려졌습니다. DFS의 체계적 정립은 1972년 로버트 타잔(Robert Tarjan)이 강한 연결 요소 알고리즘과 함께 정리한 것이 결정적이었습니다.

## 2. DFS — 재귀로 쓸 것인가, 스택으로 쓸 것인가

> **개요** DFS는 재귀로 쓰면 극도로 짧아지지만, 재귀 깊이가 곧 공간복잡도이자 사고의 원인이 됩니다.

```python
def dfs_recursive(graph, v, visited):
    visited.add(v)
    process(v)                       # 전위 처리
    for nxt in graph[v]:
        if nxt not in visited:
            dfs_recursive(graph, nxt, visited)
    finish(v)                        # 후위 처리 — 위상 정렬의 핵심
```

재귀 DFS의 매력은 **전위 처리와 후위 처리를 자연스럽게 구분**할 수 있다는 점입니다. 재귀 호출 전에 하는 일(발견 시점)과 모든 자식을 끝낸 뒤 하는 일(종료 시점)이 코드에서 위아래로 나뉘고, 이 구분이 위상 정렬과 사이클 탐지의 핵심 도구가 됩니다.

반면 반복 구현은 재귀 깊이 문제에서 자유롭습니다. 정점이 10만 개인 사슬 형태 그래프에 재귀 DFS를 쓰면 파이썬 기본 재귀 한도(1000)에 즉시 걸리고, 한도를 올려도 시스템 스택이 터집니다.

| 구현 | 장점 | 단점 |
|---|---|---|
| 재귀 | 코드가 짧고 전/후위 처리 구분이 자연스러움 | 재귀 깊이 = $O(V)$ 스택. 오버플로 위험 |
| 명시적 스택 | 깊이 제한 없음, 힙 메모리 사용 | 후위 처리 구현이 번거로움 |

> **팁** 대회나 실무에서 "DFS가 이상하게 죽는다"의 원인은 열에 아홉 재귀 깊이입니다. 정점 수가 수만을 넘길 가능성이 있으면 처음부터 명시적 스택으로 쓰거나, 재귀 한도를 명시적으로 올리고 스택 크기까지 조정하세요.

## 3. 방문 표시 — 언제 표시하느냐가 정확성을 가른다

> **Q.** 방문 표시를 큐에서 꺼낼 때 하면 안 되나요?
> **A.** BFS에서는 **반드시 큐에 넣을 때** 표시해야 합니다. 꺼낼 때 표시하면 같은 정점이 여러 번 큐에 들어가고, 정점 하나가 $\deg(v)$번까지 중복 삽입되어 최악의 경우 큐 크기와 실행 시간이 폭발합니다. 특히 완전 그래프에서는 $O(V^2)$으로 무너집니다.

방문 표시(visited)가 없으면 사이클이 있는 그래프에서 무한 루프에 빠집니다. 표시는 "이 정점은 이미 처리했거나 처리 예정이므로 다시 큐에 넣지 않는다"는 약속입니다.

```python
from collections import deque

def bfs(graph, start):
    dist = {start: 0}
    q = deque([start])
    while q:
        v = q.popleft()
        for nxt in graph[v]:
            if nxt not in dist:      # 넣을 때 즉시 기록 — 중복 삽입 차단
                dist[nxt] = dist[v] + 1
                q.append(nxt)
    return dist
```

> **해석** 위 코드에서 `dist` 딕셔너리가 방문 표시와 거리 기록을 겸합니다. `nxt not in dist` 검사와 `dist[nxt] = ...` 기록이 같은 블록 안에 붙어 있는 것이 핵심입니다. 이 둘이 떨어지면(예: 꺼낼 때 기록) 중복 삽입이 발생합니다.

**복잡도 도출.** 각 정점은 정확히 한 번 큐에 들어가고 한 번 꺼내집니다 → $O(V)$. 정점을 꺼낼 때마다 그 정점의 인접 리스트를 한 번씩 훑으므로, 전체를 합치면 모든 간선을 한 번(무방향이면 두 번) 봅니다 → $O(E)$. 따라서

$$T(V, E) = O(V + E)$$

> **참고** 이 $O(V+E)$는 **인접 리스트**로 그래프를 표현했을 때의 값입니다. 인접 행렬을 쓰면 정점마다 $V$칸을 훑어야 해서 $O(V^2)$가 됩니다. 간선이 희소한($E \ll V^2$) 실제 그래프에서 인접 리스트가 표준인 이유입니다. 그래프 표현 방식의 자세한 비교는 Part 15를 참고하세요.

## 4. BFS가 무가중치 최단 경로를 주는 이유

> **개요** "BFS는 최단 경로를 구한다"를 외우는 대신, 왜 그런지 한 줄로 증명할 수 있어야 응용이 됩니다.

핵심은 다음 <u>**불변식(invariant)**</u>입니다.

> **BFS 큐 불변식**: BFS 실행 중 어느 시점에나, 큐 안의 정점들의 거리 값은 최대 두 종류($d$와 $d+1$)뿐이고, 앞쪽이 $d$, 뒤쪽이 $d+1$인 비내림차순으로 정렬되어 있다.

![BFS의 층별 확장과 최단 경로 보장](https://raw.githubusercontent.com/SeongminJaden/dev-curriculum-assets/main/lectures/part16_알고리즘_심화/16-4_DFS와_BFS/assets/diagram2_bfs_shortest_path.svg)

> **해석** 그림처럼 BFS는 거리 0인 정점, 거리 1인 정점들, 거리 2인 정점들을 **층 단위로 완전히 소진한 뒤** 다음 층으로 넘어갑니다. 따라서 어떤 정점 $u$가 처음 발견되어 `dist[u] = d`로 기록되는 순간, 거리가 $d$ 미만인 정점들은 이미 전부 처리가 끝난 상태입니다. 만약 $u$에 이르는 더 짧은 경로 $d' < d$가 존재했다면 그 경로의 마지막 직전 정점은 거리 $d'-1$을 가지고 있었을 테고, 그 정점을 처리하는 시점에 이미 $u$를 발견했어야 합니다. 모순이므로 $d$가 최단 거리입니다.

이 성질이 성립하는 조건은 **모든 간선의 가중치가 같다**는 것입니다. 가중치가 다르면 "간선 수가 적은 경로"와 "비용이 작은 경로"가 달라지므로 BFS는 틀린 답을 냅니다. 그때는 다익스트라가 필요하며 16-7에서 다룹니다.

| 상황 | 알맞은 알고리즘 |
|---|---|
| 모든 간선 가중치 동일 | BFS — $O(V+E)$ |
| 가중치가 0 또는 1 | 0-1 BFS(덱 사용) — $O(V+E)$ |
| 양수 가중치 | 다익스트라 — $O((V+E)\log V)$ |
| 음수 가중치 존재 | 벨만-포드 — $O(VE)$ |

> **팁** "미로에서 최소 이동 횟수", "친구 관계에서 몇 다리 건너면 아는 사이인가", "체스판에서 나이트의 최소 이동" 같은 문제는 전부 무가중치 최단 경로이고, BFS가 정답입니다. **"최소 횟수"라는 단어를 보면 BFS를 먼저 떠올리세요.**

## 5. 활용 1 — 연결 요소와 이분 그래프 판정

<u>**연결 요소(connected component)**</u>는 서로 도달 가능한 정점들의 최대 집합입니다. 모든 정점을 순회하며 아직 방문하지 않은 정점에서 탐색을 시작하고, 그 횟수를 세면 연결 요소의 개수가 됩니다.

```python
def count_components(graph, n):
    visited = set()
    count = 0
    for v in range(n):
        if v not in visited:
            count += 1
            dfs_recursive(graph, v, visited)   # BFS로 바꿔도 결과 동일
    return count
```

> **해석** 이 문제에서는 DFS와 BFS 중 무엇을 써도 **결과가 완전히 같습니다.** "어떤 정점들이 서로 도달 가능한가"만 묻고 순서나 거리를 묻지 않기 때문입니다. 반대로 말하면, 순서나 거리가 중요한 문제에서만 둘 중 하나를 골라야 합니다.

<u>**이분 그래프(bipartite graph)**</u> 판정도 같은 뼈대입니다. BFS로 퍼져 나가며 인접 정점에 반대 색을 칠하고, 이미 칠해진 정점이 같은 색이면 홀수 길이 사이클이 있다는 뜻이므로 이분 그래프가 아닙니다.

## 6. 활용 2 — 사이클 탐지와 위상 정렬

> **개요** 여기서부터는 DFS와 BFS의 선택이 결과를 가릅니다.

**무방향 그래프의 사이클 탐지**는 간단합니다. DFS 중 이미 방문한 정점을 만났는데 그것이 부모가 아니라면 사이클입니다.

**방향 그래프의 사이클 탐지**는 더 미묘합니다. "이미 방문함"만으로는 부족하고, **현재 재귀 경로 위에 있는가**를 따로 구분해야 합니다. 정점을 세 가지 상태로 관리합니다.

| 상태 | 의미 |
|---|---|
| 흰색(0) | 아직 방문 안 함 |
| 회색(1) | 방문했고 현재 재귀 스택 위에 있음 |
| 검은색(2) | 모든 자식 탐색 완료 |

회색 정점으로 향하는 간선(<u>**역방향 간선, back edge**</u>)을 만나면 사이클입니다. 검은색 정점으로 향하는 간선은 이미 끝난 다른 갈래를 가리킬 뿐이므로 사이클이 아닙니다. 이 구분이 "재귀 스택 위에 있는가"를 아는 DFS에서만 자연스럽게 되는 이유입니다.

<u>**위상 정렬(topological sort)**</u>은 방향 비순환 그래프(DAG)의 정점을 "모든 간선이 앞→뒤 방향이 되도록" 나열하는 것입니다. 두 가지 표준 방법이 있습니다.

- **DFS 기반**: 후위 처리 시점(모든 자식 탐색 완료)에 정점을 리스트에 넣고, 마지막에 뒤집습니다. 자식이 모두 끝난 뒤에 자신을 넣으므로 자연히 자신이 앞에 옵니다.
- **칸(Kahn) 알고리즘 — BFS 기반**: 진입 차수가 0인 정점을 큐에 넣고, 꺼낼 때마다 그 정점에서 나가는 간선을 제거해 진입 차수를 줄이며, 0이 된 정점을 큐에 넣습니다. 1962년 아서 칸(Arthur B. Kahn)이 발표했습니다.

```python
from collections import deque

def topo_sort_kahn(graph, indeg, n):
    q = deque(v for v in range(n) if indeg[v] == 0)
    order = []
    while q:
        v = q.popleft()
        order.append(v)
        for nxt in graph[v]:
            indeg[nxt] -= 1
            if indeg[nxt] == 0:
                q.append(nxt)
    return order if len(order) == n else None   # 길이가 모자라면 사이클 존재
```

> **해석** 마지막 줄이 칸 알고리즘의 숨은 보너스입니다. 결과 리스트의 길이가 정점 수보다 짧다면 진입 차수가 끝내 0이 되지 못한 정점들이 남았다는 뜻이고, 그것은 곧 **사이클이 존재한다**는 증거입니다. 위상 정렬과 사이클 탐지를 한 번의 순회로 동시에 해결합니다.

> **참고** 위상 정렬은 실무에서 매일 돌아갑니다. 빌드 도구가 모듈 컴파일 순서를 정할 때, 패키지 매니저가 의존성 설치 순서를 정할 때, 스프레드시트가 수식 재계산 순서를 정할 때, 태스크 스케줄러가 작업 순서를 정할 때 모두 위상 정렬입니다. 그리고 "순환 의존성 오류"라는 익숙한 메시지는 바로 위 코드의 마지막 줄이 `None`을 반환한 결과입니다.

## 7. 어느 쪽을 골라야 하는가

| 문제 | 선택 | 이유 |
|---|---|---|
| 무가중치 최단 경로 / 최소 이동 횟수 | BFS | 층 단위 확장이 최단 거리를 보장 |
| 연결 요소 개수 / 도달 가능성 | 둘 다 가능 | 순서·거리가 무관 |
| 방향 그래프 사이클 탐지 | DFS | 재귀 스택 위 여부(회색)를 알아야 함 |
| 위상 정렬 | 둘 다 가능 | DFS 후위 처리 / 칸 알고리즘 |
| 모든 경로 열거 / 백트래킹 | DFS | 되돌아오며 상태를 복구하기 좋음 |
| 상태 공간이 매우 넓고 답이 얕음 | BFS | 깊이 폭주 없이 얕은 답을 먼저 찾음 |
| 메모리가 빠듯한 깊은 그래프 | DFS | BFS는 한 층 전체를 큐에 담아야 함 |

> **해석** 마지막 두 줄이 트레이드오프를 잘 보여줍니다. BFS는 최단성을 보장하는 대신 **한 층 전체**를 메모리에 올려야 해서, 분기 계수가 큰 그래프에서 큐가 폭발합니다. DFS는 메모리를 깊이만큼만 쓰지만 최단성을 보장하지 않고 깊은 갈래에 빠질 수 있습니다. 두 장점을 절충한 것이 깊이 제한을 1씩 늘려가며 DFS를 반복하는 **반복 심화 탐색(IDDFS)** 으로, 게임 트리 탐색에서 널리 쓰입니다.

---

## 요약

> **핵심**
> - DFS와 BFS의 코드 차이는 후보 정점을 담는 통에서 무엇을 먼저 꺼내느냐, 즉 **스택이냐 큐냐** 한 줄뿐이지만, 그 결과 방문 순서가 세로(깊이)와 가로(층)로 갈립니다.
> - 방문 표시는 사이클에서의 무한 루프를 막는 장치이며, BFS에서는 **큐에 넣는 시점에** 표시해야 중복 삽입이 방지됩니다. 인접 리스트 기준 두 탐색 모두 $O(V+E)$입니다.
> - BFS는 층을 완전히 소진하고 다음 층으로 넘어가므로 정점이 처음 발견되는 순간의 거리가 곧 최단 거리입니다. 단 **모든 간선 가중치가 같을 때만** 성립합니다.
> - 연결 요소는 둘 다 가능하지만, 방향 그래프 사이클 탐지는 재귀 스택 위 여부(회색 상태)를 아는 DFS가 자연스럽고, 위상 정렬은 DFS 후위 처리와 칸 알고리즘 두 표준 해법이 있으며 칸 알고리즘은 사이클 탐지를 덤으로 줍니다.
