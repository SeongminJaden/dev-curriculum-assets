# 16-2. 정렬 알고리즘 — 실습

> **참고** 이 실습은 제출물이 아니라, 정렬 알고리즘의 성질을 직접 구현하고 측정해 눈으로 확인하는 과정입니다.

## 실습 1 — 세 가지 정렬을 구현하고 연산 횟수 세기

- [ ] 버블·선택·삽입 정렬을 각각 구현하되, 비교 횟수와 교환 횟수를 세는 카운터를 넣으세요.
- [ ] 크기 1,000인 (a) 무작위 배열, (b) 이미 정렬된 배열, (c) 역순 배열 세 가지로 각 정렬을 돌려 카운터 값을 9칸짜리 표로 정리하세요.
- [ ] 선택 정렬의 교환 횟수가 입력 종류와 무관하게 항상 999인지 확인하고, 왜 그런지 한 문장으로 적어보세요.
- [ ] 삽입 정렬이 (b)에서만 압도적으로 빠른 이유를 "뒤집힌 쌍(inversion)"이라는 단어를 써서 설명하세요.

## 실습 2 — 안정성을 직접 깨뜨려보기

- [ ] `(이름, 점수)` 쌍 10개를 만들되 점수가 같은 쌍이 3개 이상 포함되게 하세요.
- [ ] 점수를 키로 삽입 정렬을 돌리고, 같은 점수끼리 원래 입력 순서가 보존되는지 출력해 확인하세요.
- [ ] 삽입 정렬 코드의 `a[j] > key` 조건을 `a[j] >= key`로 바꾼 뒤 같은 입력으로 다시 돌려, 같은 점수의 순서가 뒤집히는 것을 확인하세요.
- [ ] 사용하는 언어의 표준 정렬 함수가 안정인지 불안정인지 같은 방법으로 실험해 판정하세요.

## 실습 3 — 퀵 정렬의 최악을 재현하고 막기

- [ ] 항상 첫 원소를 피벗으로 쓰는 퀵 정렬을 구현하고, 재귀 깊이를 기록하는 변수를 추가하세요.
- [ ] 무작위 배열 5,000개와 이미 정렬된 배열 5,000개를 각각 넣어 재귀 깊이와 실행 시간을 비교하세요.
- [ ] 피벗 선택을 "첫·중간·끝의 중앙값"으로 바꾼 뒤 정렬된 배열로 다시 측정하고, 깊이가 몇 배 줄었는지 계산하세요.
- [ ] 재귀 깊이가 $2\log_2 n$을 넘으면 힙 정렬로 전환하는 introsort 방식을 추가해, 최악 입력에서도 깊이가 제한되는지 확인하세요.

## 실습 4 — 계수 정렬과 표준 정렬의 손익분기점 찾기

- [ ] 값 범위가 0~99인 정수 100만 개를 만들어 계수 정렬과 표준 정렬 함수의 실행 시간을 비교하세요.
- [ ] 값 범위를 0~1,000,000으로 바꿔 같은 실험을 반복하고, 계수 정렬의 메모리 사용량과 시간이 어떻게 변하는지 기록하세요.
- [ ] 두 실험 결과를 근거로 "계수 정렬을 쓸 만한 조건"을 $n$과 $k$의 관계식으로 적어보세요.

> **팁** 실습 3에서 정렬된 배열이 퀵 정렬의 최악이 되는 장면은 반드시 직접 재현해보세요. 실무 사고의 상당수가 "테스트에서는 무작위 데이터를 썼는데 운영에서는 이미 정렬된 데이터가 들어왔다"에서 발생합니다.
