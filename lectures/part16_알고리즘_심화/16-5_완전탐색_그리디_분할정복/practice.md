# 16-5. 완전탐색·그리디·분할정복 — 실습

> **참고** 이 실습은 제출물이 아니라, 그리디의 반례를 직접 찾아내고 가지치기의 효과를 실측해보는 과정입니다.

## 실습 1 — 상태 공간 크기 계산하고 완전탐색 짜기

- [ ] 정수 $n$개에서 부분집합을 골라 합이 목표값이 되는 경우를 찾는 코드를 비트마스크 완전탐색으로 구현하세요.
- [ ] $n$을 15, 20, 25로 늘려가며 실행 시간을 측정해 표로 기록하고, $2^n$이 예측한 배율과 맞는지 확인하세요.
- [ ] 같은 코드에 "현재 합이 이미 목표를 넘으면 중단"하는 유망성 가지치기를 넣고 $n=25$에서 시간이 얼마나 줄었는지 비교하세요.
- [ ] $n$개 도시를 모두 도는 외판원 문제를 순열 완전탐색으로 짜고, $n$이 몇일 때 1초를 넘기는지 직접 찾아 기록하세요.

## 실습 2 — 그리디의 반례를 직접 찾아내기

- [ ] 동전 종류 목록과 목표 금액을 받아 큰 동전부터 쓰는 그리디 거스름돈 함수를 구현하세요.
- [ ] 같은 문제를 완전탐색(또는 DP)으로 정확히 푸는 함수도 구현하세요.
- [ ] 동전 종류를 무작위로 3~4개 생성하고 목표 금액을 1~50으로 바꿔가며 두 함수를 대조해, 결과가 다른 사례를 최소 3개 수집하세요.
- [ ] 수집한 반례에서 그리디의 첫 선택이 어디서 잘못되었는지 각각 한 줄로 적으세요.

## 실습 3 — 그리디가 성립하는 문제 증명해보기

- [ ] (시작, 종료) 시각 쌍 20개를 만들고, 종료 시각 기준 정렬 그리디로 최대 회의 개수를 구하세요.
- [ ] 시작 시각 기준 정렬, 회의 길이 기준 정렬 두 가지 다른 그리디도 구현해 세 결과를 비교하고, 각각의 반례를 하나씩 만들어보세요.
- [ ] 종료 시각 기준 그리디가 옳은 이유를 교환 논법으로 서술하세요("최적해 $S$가 그리디 선택 $g$를 포함하지 않는다고 가정하면…" 형식으로 5줄 이내).

## 실습 4 — 분할 정복 구현과 점화식 검증

- [ ] 빠른 거듭제곱 `power(base, exp, mod)`를 재귀로 구현하고, `exp`가 $10^{18}$일 때 곱셈 호출 횟수를 세어 $\log_2 exp$와 비교하세요.
- [ ] 같은 계산을 단순 반복 곱셈으로도 시도해 실행 시간 차이를 측정하세요(작은 지수부터 늘려가며).
- [ ] 배열에서 최댓값·최솟값을 동시에 찾는 문제를 (a) 단순 반복과 (b) 분할 정복으로 구현하고, 비교 횟수를 세어 분할 정복이 약 $1.5n$번임을 확인하세요.
- [ ] 병합 정렬과 퀵 정렬의 "분할 비용"과 "결합 비용"을 각각 계측하는 카운터를 넣어, 두 알고리즘이 서로 반대쪽에 비용을 몰아넣었음을 수치로 보이세요.

> **팁** 실습 2가 이 강의의 핵심입니다. 그리디는 코드가 짧아서 맞아 보이고, 틀려도 예외를 던지지 않습니다. "완전탐색으로 먼저 짜고 대조한다"는 습관을 여기서 몸에 익히면, 앞으로 그리디를 떠올릴 때마다 자동으로 반례를 찾게 됩니다.
