# Part 16. 알고리즘 심화

## 16-5. 완전탐색·그리디·분할정복 — 언제 "일단 다 해보기"를 버려도 되는가

> **학습 목표**
> - 완전탐색의 상태 공간 크기를 계산하고, 가지치기로 실행 시간을 줄이는 방법을 적용할 수 있다.
> - 그리디 알고리즘의 정당성 조건인 탐욕 선택 속성과 최적 부분 구조를 설명하고, 반례를 들어 그리디가 실패하는 상황을 판별할 수 있다.
> - 분할 정복의 세 단계 구조를 설명하고, 대표 알고리즘에 적용해 점화식과 복잡도를 도출할 수 있다.

> **개요** 모든 알고리즘 설계는 "모든 경우를 다 해본다"는 완전탐색에서 출발합니다. 그리고 대부분의 발전은 **다 해보지 않아도 되는 이유를 찾는 과정**입니다. 그리디는 "매 순간 최선을 고르면 전체 최선이 된다"는 이유를, 분할 정복은 "문제를 쪼개면 합칠 수 있다"는 이유를 찾은 것입니다. 이 강의는 세 전략을 한 흐름으로 놓고, 특히 그리디가 언제 정당하고 언제 조용히 틀린 답을 내는지에 집중합니다.

## 1. 완전탐색 — 가장 정직하고 가장 확실한 출발점

> **Q.** 완전탐색은 느리니까 처음부터 더 좋은 방법을 찾아야 하지 않나요?
> **A.** 반대입니다. 완전탐색은 **정답이 보장되는 유일한 기준선**입니다. 실무와 대회 양쪽에서 표준 절차는 이렇습니다. (1) 완전탐색으로 확실히 맞는 코드를 먼저 짠다. (2) 상태 공간 크기를 계산해 입력 제한 안에 들어오는지 본다. (3) 안 들어오면 그때 가지치기·그리디·DP로 개선하되, 완전탐색 결과와 대조해 검증한다. 3단계의 대조 검증을 건너뛰면 "빠르지만 틀린 코드"가 나옵니다.

<u>**완전탐색(brute force / exhaustive search)**</u>은 가능한 모든 경우를 빠짐없이 만들어 검사합니다. 설계의 핵심은 "경우를 어떻게 빠짐없이, 중복 없이 생성할 것인가"입니다.

| 경우의 수 형태 | 크기 | 대표 문제 |
|---|---|---|
| 각 원소를 넣거나 뺌(부분집합) | $2^n$ | 부분수열 합, 비트마스크 |
| 순서를 정함(순열) | $n!$ | 외판원, 배치 문제 |
| $n$개 중 $r$개 고름(조합) | $\binom{n}{r}$ | 팀 나누기 |
| 각 칸에 $k$가지 중 하나(중복순열) | $k^n$ | 암호 대입, 색칠 |
| 좌표/상태 공간 순회 | 상태 수 | 미로, 시뮬레이션 |

> **해석** 표의 첫 두 줄이 실전에서 가장 자주 나옵니다. 16-1의 허용 한계표를 떠올리면 $2^n$은 $n \le 25$, $n!$은 $n \le 11$ 정도가 한계입니다. **문제를 읽자마자 이 계산부터 하면 어떤 전략이 필요한지가 대부분 결정됩니다.**

## 2. 가지치기 — 완전탐색을 살려내는 기술

> **개요** <u>**가지치기(pruning)**</u>는 탐색 트리에서 "이 갈래로는 절대 정답이 나올 수 없다"고 판정되는 순간 그 아래 전체를 건너뛰는 것입니다. 점근 복잡도는 그대로지만 실측 성능은 수백 배 달라질 수 있습니다.

![완전탐색 트리와 가지치기](https://raw.githubusercontent.com/SeongminJaden/dev-curriculum-assets/main/lectures/part16_알고리즘_심화/16-5_완전탐색_그리디_분할정복/assets/diagram1_bruteforce_pruning.svg)

> **해석** 그림에서 회색으로 처리된 부분이 가지치기로 잘려나간 영역입니다. 트리의 위쪽에서 자를수록 잘리는 노드 수가 기하급수적으로 많아진다는 점이 중요합니다. 깊이 $d$에서 한 갈래를 자르면 그 아래 $b^{h-d}$개의 노드가 통째로 사라집니다. **그래서 가지치기 조건은 되도록 얕은 곳에서 판정할 수 있게 설계해야 합니다.**

대표적인 가지치기 유형은 셋입니다.

- **유망성 검사(feasibility)**: 현재 부분해가 이미 제약을 위반했다면 중단. 예) 배낭 무게 초과, N-Queens에서 이미 공격당함.
- **한계값 검사(bounding)**: 현재 부분해에서 얻을 수 있는 **최선의 값**을 계산해 지금까지의 최적해보다 나쁘면 중단. 이것을 체계화한 것이 <u>**분기 한정법(branch and bound)**</u>입니다.
- **대칭성 제거**: 본질적으로 같은 경우를 한 번만 세기. 예) N-Queens에서 첫 행의 위치를 절반만 시도.

```python
best = float('inf')

def search(i, cost, used):
    global best
    if cost >= best:              # 한계값 가지치기 — 이 줄 하나가 수백 배를 만듦
        return
    if i == n:
        best = cost
        return
    for j in range(n):
        if not used[j]:
            used[j] = True
            search(i + 1, cost + w[i][j], used)
            used[j] = False       # 백트래킹 — 상태 복구
```

> **참고** 위 코드에서 `if cost >= best: return` 한 줄을 빼면 $n!$개를 전부 탐색합니다. 넣으면 대부분의 실제 입력에서 훨씬 일찍 끝납니다. 그런데 **최악의 경우 복잡도는 여전히 $O(n!)$입니다.** 가지치기는 점근 복잡도를 낮추지 못한다는 사실을 기억하세요. 점근 복잡도를 실제로 낮추려면 DP처럼 구조적인 재설계가 필요합니다(16-6 참고).

> **팁** 한계값 가지치기의 효과는 "좋은 해를 얼마나 빨리 찾느냐"에 크게 좌우됩니다. `best`가 일찍 작아질수록 잘리는 갈래가 많아지므로, 탐색 순서를 유망한 후보부터 시도하도록 정렬하는 것이 큰 도움이 됩니다.

## 3. 그리디 — 매 순간의 최선이 전체 최선이 되는 조건

> **Q.** 그리디는 "그냥 좋아 보이는 것을 고른다"는 건데, 어떻게 최적해가 보장되나요?
> **A.** 보장되지 않습니다. **오직 문제가 두 가지 성질을 만족할 때만** 보장됩니다. 그리디의 어려움은 알고리즘을 짜는 데 있지 않고(대개 정렬 한 번 + 반복문 한 번입니다), **그 두 성질이 성립하는지 증명하는 데** 있습니다.

<u>**그리디 알고리즘(greedy algorithm)**</u>이 최적해를 내려면 다음 두 조건이 필요합니다.

- <u>**탐욕 선택 속성(greedy choice property)**</u>: 지금 국소적으로 최선인 선택을 포함하는 전체 최적해가 반드시 존재한다. 즉 "지금의 최선"을 골라도 최적해를 잃지 않는다.
- <u>**최적 부분 구조(optimal substructure)**</u>: 하나를 선택한 뒤 남은 부분 문제의 최적해와 그 선택을 합치면 전체 최적해가 된다.

**성공 사례 — 회의실 배정.** 시간이 겹치지 않게 최대한 많은 회의를 배정하는 문제입니다. **끝나는 시각이 이른 것부터** 고르면 최적입니다.

> **해석** 탐욕 선택 속성의 증명은 **교환 논법(exchange argument)** 으로 합니다. 어떤 최적해 $S$가 우리의 그리디 선택 $g$(가장 일찍 끝나는 회의)를 포함하지 않는다고 합시다. $S$의 첫 회의를 $s$라 하면 $g$의 종료 시각은 $s$ 이하입니다. 따라서 $S$에서 $s$를 빼고 $g$를 넣어도 나머지 회의들과 여전히 겹치지 않고, 개수도 같습니다. 즉 $g$를 포함하는 최적해가 존재합니다. **"그리디 선택으로 바꿔치기해도 손해가 없다"를 보이는 것**이 교환 논법의 전부입니다.

**실패 사례 — 동전 거스름돈.** 큰 동전부터 욕심껏 쓰는 방식은 동전 체계에 따라 틀립니다.

| 동전 체계 | 목표 금액 | 그리디 결과 | 실제 최적 |
|---|---|---|---|
| 1, 10, 100, 500 (한국) | 620 | 500+100+10+10 = 4개 | 4개 (일치) |
| 1, 4, 6 | 8 | 6+1+1 = 3개 | 4+4 = **2개** |
| 1, 3, 4 | 6 | 4+1+1 = 3개 | 3+3 = **2개** |

> **해석** 아래 두 줄이 그리디의 위험을 압축해 보여줍니다. 코드는 정상 실행되고 그럴듯한 답을 내지만 **최적이 아닙니다.** 그리고 예외가 발생하지 않기 때문에 테스트를 통과할 수도 있습니다. 한국·미국 통화가 그리디로 잘 되는 것은 우연이 아니라 각 동전이 그 아래 동전의 배수 관계로 설계되었기 때문입니다. 이런 체계를 canonical coin system이라 부르며, 임의의 체계는 DP로 풀어야 합니다.

![그리디가 성공하는 경우와 실패하는 경우](https://raw.githubusercontent.com/SeongminJaden/dev-curriculum-assets/main/lectures/part16_알고리즘_심화/16-5_완전탐색_그리디_분할정복/assets/diagram2_greedy_vs_optimal.svg)

> **해석** 그림 왼쪽 회의실 배정에서는 그리디 경로가 최적 경로와 정확히 겹칩니다. 오른쪽 거스름돈에서는 첫 선택(6원)에서 갈라져 나와 다시는 최적 경로로 돌아오지 못합니다. 그리디는 **한 번 내린 결정을 되돌리지 않기 때문에**, 초기의 잘못된 선택이 끝까지 남습니다. 되돌릴 수 있게 만든 것이 백트래킹이고, 모든 선택지를 기억해 두는 것이 DP입니다.

> **참고** 실무에서 유명한 그리디 알고리즘으로는 1952년 데이비드 허프먼(David A. Huffman)이 MIT 학생 시절 발표한 허프먼 코딩이 있습니다. 빈도가 낮은 두 기호를 계속 합쳐 올라가는 그리디 전략으로 최적 접두 부호를 만들며, ZIP·JPEG·MP3의 엔트로피 부호화 단계에 지금도 쓰입니다. 최소 신장 트리의 크루스칼·프림도 그리디이며 16-7에서 다룹니다.

> **팁** 그리디를 떠올렸다면 반드시 **반례를 5분 동안 찾아보세요.** 작은 입력을 만들어 완전탐색 결과와 대조하는 것이 가장 빠릅니다. 반례를 못 찾으면 교환 논법으로 증명을 시도하고, 증명도 안 되면 DP로 가는 것이 안전합니다.

## 4. 분할 정복 — 쪼개고, 풀고, 합치기

> **개요** <u>**분할 정복(divide and conquer)**</u>은 문제를 같은 형태의 작은 문제로 나눠 재귀적으로 풀고 결과를 합칩니다. 세 단계로 정형화됩니다.

1. **분할(Divide)**: 문제를 같은 종류의 부분 문제로 나눕니다.
2. **정복(Conquer)**: 부분 문제를 재귀적으로 풉니다. 충분히 작아지면 직접 풉니다.
3. **결합(Combine)**: 부분 해를 합쳐 전체 해를 만듭니다.

복잡도는 $T(n) = a\,T(n/b) + f(n)$ 형태의 점화식이 되고, 마스터 정리로 풉니다(16-1 참고).

| 알고리즘 | 분할 | 결합 | 점화식 | 복잡도 |
|---|---|---|---|---|
| 병합 정렬 | 절반으로 나눔(쉬움) | 정렬된 두 배열 병합(어려움) | $2T(n/2)+\Theta(n)$ | $\Theta(n \log n)$ |
| 퀵 정렬 | 피벗 기준 분할(어려움) | 아무것도 안 함(쉬움) | $2T(n/2)+\Theta(n)$ | $\Theta(n \log n)$ 평균 |
| 이진 탐색 | 절반 버림 | 없음 | $T(n/2)+\Theta(1)$ | $\Theta(\log n)$ |
| 거듭제곱 분할 | 지수를 반으로 | 제곱 한 번 | $T(n/2)+\Theta(1)$ | $\Theta(\log n)$ |
| 카라추바 곱셈 | 자릿수 절반 | 덧셈 몇 번 | $3T(n/2)+\Theta(n)$ | $\Theta(n^{1.585})$ |
| 최근접 점 쌍 | 좌우로 나눔 | 경계 근처 검사 | $2T(n/2)+\Theta(n)$ | $\Theta(n \log n)$ |

> **해석** 병합 정렬과 퀵 정렬을 나란히 보면 분할 정복의 **비용 총량 보존** 성질이 드러납니다. 병합 정렬은 분할이 공짜인 대신 결합이 비싸고, 퀵 정렬은 분할이 비싼 대신 결합이 공짜입니다. 어느 쪽에 일을 몰아넣을 것인가가 설계 선택이며, 그 선택이 안정성·제자리 여부 같은 부수적 성질까지 결정합니다(16-2 참고).

**빠른 거듭제곱**은 분할 정복의 가장 짧은 예입니다.

```python
def power(base, exp, mod):
    if exp == 0:
        return 1
    half = power(base, exp // 2, mod)
    result = half * half % mod
    if exp % 2:
        result = result * base % mod
    return result
```

> **해석** $a^n$을 $n$번 곱하는 대신 $a^{n/2}$를 구해 제곱합니다. $\Theta(n)$이 $\Theta(\log n)$이 되며, $n$이 $10^{18}$이어도 60번의 곱셈으로 끝납니다. RSA 암호의 모듈러 거듭제곱이 실용적인 속도로 돌아가는 것이 바로 이 알고리즘 덕분입니다.

> **참고** 1960년 아나톨리 카라추바(Anatoly Karatsuba)는 23세 때 $n$자리 수 곱셈이 $\Theta(n^2)$보다 빠를 수 없다는 콜모고로프의 추측을 일주일 만에 반증했습니다. 두 수를 절반으로 나눠 곱셈 4번이 필요해 보이는 것을 대수적 항등식으로 3번으로 줄여 $\Theta(n^{\log_2 3}) = \Theta(n^{1.585})$을 얻었습니다. 슈트라센의 행렬 곱과 정확히 같은 구조의 발상이며(16-1 참고), 파이썬의 큰 정수 곱셈도 자릿수가 일정 이상이면 카라추바로 전환합니다.

## 5. 세 전략의 관계 — 하나의 스펙트럼

> **개요** 완전탐색·그리디·분할 정복(그리고 다음 강의의 DP)은 별개의 기법이 아니라 **"얼마나 많은 경우를 실제로 살펴봐야 하는가"라는 하나의 축** 위에 놓입니다.

| 전략 | 부분 문제를 몇 개 보는가 | 되돌릴 수 있는가 | 최적 보장 |
|---|---|---|---|
| 완전탐색 | 전부 | — | 항상 |
| 백트래킹 | 유망한 것만 | 예 | 항상 |
| 분할 정복 | 겹치지 않는 부분 문제 전부 | — | 항상 |
| 동적 계획법 | 겹치는 부분 문제를 한 번씩 | — | 항상 |
| 그리디 | 각 단계에서 하나만 | 아니오 | 조건 만족 시에만 |

> **해석** 아래로 갈수록 살펴보는 경우가 줄어 빨라지지만, 마지막 줄에서 처음으로 **최적 보장이 조건부**가 됩니다. 이것이 그리디만 유독 증명을 요구하는 이유입니다. 또한 분할 정복과 DP의 차이는 **부분 문제가 겹치는가**입니다. 병합 정렬의 좌우 절반은 서로 겹치지 않아 재귀만으로 충분하지만, 피보나치의 부분 문제는 심하게 겹쳐서 기억해 두지 않으면 지수 시간이 됩니다. 이 이야기는 16-6에서 이어집니다.

> **팁** 문제를 만났을 때의 실전 순서를 권합니다. ① 상태 공간 크기를 계산한다 → ② 완전탐색으로 들어오면 그냥 짠다 → ③ 안 들어오면 그리디가 되는지 반례를 찾아본다 → ④ 반례가 나오면 부분 문제가 겹치는지 본다 → ⑤ 겹치면 DP, 안 겹치면 분할 정복, 둘 다 아니면 가지치기를 강화한 백트래킹.

## 요약

> **핵심**
> - 완전탐색은 느리지만 **정답이 보장되는 기준선**입니다. 상태 공간 크기($2^n$, $n!$, $k^n$ 등)를 먼저 계산해 입력 제한과 대조하는 것이 모든 설계의 첫 단계입니다.
> - 가지치기는 유망성·한계값·대칭성 세 유형으로 나뉘며 **트리의 얕은 곳에서 자를수록** 효과가 기하급수적입니다. 다만 점근 복잡도 자체는 낮추지 못합니다.
> - 그리디는 **탐욕 선택 속성과 최적 부분 구조**를 모두 만족할 때만 최적해를 냅니다. 성립하지 않으면 예외 없이 조용히 틀린 답을 내므로, 반드시 반례 탐색이나 교환 논법으로 검증해야 합니다.
> - 분할 정복은 분할·정복·결합의 세 단계이며 비용은 분할과 결합 중 한쪽에 몰립니다. 부분 문제가 **겹치지 않을 때** 유효하며, 겹치면 동적 계획법의 영역입니다.
