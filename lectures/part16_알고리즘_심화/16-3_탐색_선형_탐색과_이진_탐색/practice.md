# 16-3. 탐색 — 선형 탐색과 이진 탐색 — 실습

> **참고** 이 실습은 제출물이 아니라, 이진 탐색의 경계 조건을 직접 깨뜨려보고 파라메트릭 서치를 손으로 설계해보는 과정입니다.

## 실습 1 — 경계 버그를 일부러 만들어보기

- [ ] 닫힌 구간 `[lo, hi]` 방식의 이진 탐색을 구현하고, 크기 1·2·3인 배열과 배열에 없는 값으로 각각 테스트해 정상 동작을 확인하세요.
- [ ] `lo = mid + 1`을 `lo = mid`로 바꾼 뒤 같은 테스트를 돌려, 어떤 입력에서 무한 루프가 발생하는지 기록하세요.
- [ ] `while lo <= hi`를 `while lo < hi`로 바꾼 뒤 크기 1인 배열을 넣어, 정답을 놓치는 경우를 재현하세요.
- [ ] 루프 불변식("정답이 존재한다면 `[lo, hi]` 안에 있다")을 한 문장으로 적고, 위 두 변형이 그 불변식을 어떻게 깨는지 각각 설명하세요.

## 실습 2 — lower_bound와 upper_bound 직접 만들기

- [ ] 중복 값이 포함된 정렬 배열(예: `[1,3,3,3,5,7,7,9]`)을 준비하세요.
- [ ] `lower_bound`와 `upper_bound`를 반열린 구간 `[lo, hi)` 방식으로 구현하세요.
- [ ] 값 3에 대해 두 함수의 반환값을 출력하고, 그 차이가 3의 개수와 일치하는지 확인하세요.
- [ ] 배열에 없는 값 4와 범위 밖의 값 0, 10에 대해서도 두 함수를 호출해 반환값이 무엇을 의미하는지 표로 정리하세요.
- [ ] 사용하는 언어의 표준 함수(Python `bisect_left`/`bisect_right`, C++ `lower_bound`/`upper_bound`)와 결과가 일치하는지 대조하세요.

## 실습 3 — 파라메트릭 서치 설계하기

- [ ] "길이가 제각각인 랜선 N개를 잘라 같은 길이의 랜선 K개를 만들 때 최대 길이"를 구하는 코드를 결정 함수 `can_make(length)`와 이진 탐색으로 구현하세요.
- [ ] 후보 길이를 1부터 하나씩 전부 시도하는 완전탐색 버전도 만들어, N=100,000 데이터에서 두 방식의 실행 시간을 비교하세요.
- [ ] `can_make`가 단조 함수임을 확인하기 위해 길이를 1씩 늘려가며 반환값을 출력하고, 참에서 거짓으로 바뀌는 지점이 정확히 한 곳뿐인지 확인하세요.
- [ ] "$K$번째로 작은 수 찾기"를 "$x$ 이하인 수가 $K$개 이상인가?"라는 결정 문제로 바꿔 같은 방식으로 풀어보세요.

## 실습 4 — 선형 탐색과의 손익분기점 측정

- [ ] 크기 100,000인 무작위 배열에서 탐색 횟수 $q$를 1, 10, 100, 1,000, 10,000으로 바꿔가며 (a) 선형 탐색만, (b) 한 번 정렬 후 이진 탐색 두 방식의 총 실행 시간을 측정하세요.
- [ ] 두 곡선이 교차하는 $q$ 값을 찾아 기록하고, 강의의 부등식 $n \log n + q \log n < q n$이 예측한 지점과 비교하세요.
- [ ] 회전된 정렬 배열(예: `[4,5,6,7,0,1,2]`)에 기본 이진 탐색을 적용해 틀린 답이 나오는 것을 확인하고, "어느 절반이 정렬되었는지" 판정하는 조건을 추가해 고쳐보세요.

> **팁** 실습 1을 건너뛰지 마세요. 이진 탐색은 "돌아가는 코드"와 "정확한 코드" 사이의 간격이 가장 큰 알고리즘 중 하나입니다. 버그를 일부러 만들어 무한 루프를 직접 겪어보면 이후로는 불변식을 먼저 적는 습관이 생깁니다.
