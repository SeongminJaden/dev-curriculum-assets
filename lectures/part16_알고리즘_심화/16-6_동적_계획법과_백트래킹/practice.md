# 16-6. 동적 계획법과 백트래킹 — 실습

> **참고** 이 실습은 제출물이 아니라, 점화식을 직접 세우고 top-down·bottom-up 두 방식으로 구현해 보며 DP와 백트래킹의 감각을 익히는 과정입니다.

## 실습 1 — 피보나치로 중복 부분 문제 체감하기

- [ ] 메모이제이션 없이 순수 재귀로 `fib(n)`을 구현하고, `fib(35)`, `fib(40)`의 실행 시간을 측정해보세요.
- [ ] 같은 함수에 호출 카운터를 넣어 `fib(20)`을 계산할 때 `fib(5)`가 몇 번 호출되는지 세어보세요.
- [ ] 메모이제이션을 추가한 버전과 실행 시간·호출 횟수를 비교하고, $O(2^n)$이 $O(n)$으로 바뀌는 것을 수치로 확인해보세요.

## 실습 2 — 0/1 배낭 문제 점화식 구현

- [ ] 무게 한도 $W=10$, 물건 5개(무게·가치는 임의로 정함)에 대해 $dp[i][c]$ 표를 손으로 몇 칸 채워보세요.
- [ ] 점화식 $dp[i][c] = \max(dp[i-1][c],\ dp[i-1][c-w_i]+v_i)$를 bottom-up으로 구현하고, 완전탐색($2^5$가지 부분집합)의 결과와 일치하는지 대조해보세요.
- [ ] 2차원 표를 1차원으로 줄이는 공간 최적화(rolling array)를 적용하고, 이때 안쪽 반복문을 왜 역순으로 돌아야 하는지 값을 추적해 확인해보세요.

## 실습 3 — Top-down과 Bottom-up 둘 다 짜보기

- [ ] "계단 오르기"(한 번에 1칸 또는 2칸, $n$번째 계단까지 오는 경우의 수) 문제의 점화식을 세워보세요.
- [ ] 같은 점화식을 메모이제이션(재귀)과 타뷸레이션(반복) 두 방식으로 각각 구현해보세요.
- [ ] $n$을 크게 키웠을 때 재귀 버전에서 스택 오버플로가 나는 지점을 찾고, 반복 버전은 왜 그 문제가 없는지 설명해보세요.

## 실습 4 — N-Queens로 백트래킹 뼈대 익히기

- [ ] $N=4$에 대해 상태 공간 트리를 직접 그리고, 열·대각선 충돌로 가지치기되는 경로를 표시해보세요.
- [ ] 백트래킹 코드를 구현해 $N=4$부터 $N=10$까지 해의 개수를 출력하고, 알려진 값(4→2, 8→92)과 대조해보세요.
- [ ] 가지치기 조건(`continue`)을 제거해 순수 완전탐색으로 바꾼 뒤 실행 시간을 비교하고, 가지치기가 실제 탐색량을 얼마나 줄이는지 관찰해보세요.

> **팁** 실습 2·3에서 점화식이 막히면 코드를 짜기 전에 "상태 $dp[i]$가 정확히 무엇의 답인지"를 한 문장으로 먼저 적어보세요. 그 정의가 명확하지 않으면 어떤 구현도 맞지 않습니다. 실습 4의 상태 공간 트리는 손으로 그려봐야 백트래킹의 되돌리기가 눈에 들어옵니다.
