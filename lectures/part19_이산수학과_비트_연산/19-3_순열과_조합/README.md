# Part 19. 이산수학과 비트 연산

## 19-3. 순열과 조합 — 코드를 짜기 전에 "터질지" 계산할 수 있는가

> **학습 목표**
> - 순열 $P(n,r)$과 조합 $C(n,r)$의 정의를 유도하고, 문제에서 순서가 의미를 갖는지 판별해 알맞은 공식을 고를 수 있다.
> - 파스칼의 삼각형과 점화식 $C(n,r) = C(n-1,r-1) + C(n-1,r)$을 조합론적으로 설명할 수 있다.
> - 완전탐색의 경우의 수를 미리 계산해, $n!$·$2^n$·$n^2$ 중 어떤 규모에서 시간 초과가 나는지 코드를 짜기 전에 예측할 수 있다.

> **개요** 완전탐색을 구현하고 나서 시간 초과를 만나는 것과, 구현하기 전에 "이건 $12!$이니 4억 8천만이라 안 되겠군"이라고 판단하는 것은 개발 속도에서 큰 차이를 만듭니다. 이 강의는 경우의 수를 세는 수학인 순열·조합·중복 조합·비둘기집 원리를 정리하고, 그 숫자를 실행 시간 감각으로 옮기는 훈련을 합니다. 탐색 알고리즘의 구현 자체는 Part 16에서 다루므로, 여기서는 **세는 일**에 집중합니다.

## 1. 순열과 조합 — 순서가 의미를 갖는가

> **Q.** 어떤 문제에 순열을 쓰고 어떤 문제에 조합을 쓸지 매번 헷갈립니다. 확실한 판별법이 있나요?
> **A.** "골라놓은 것들의 **순서를 바꾸면 다른 답인가**"를 물어보면 됩니다. 5명 중 3명을 뽑아 회장·부회장·총무를 정한다면 (A회장, B부회장)과 (B회장, A부회장)은 다른 결과이므로 순열입니다. 5명 중 3명을 뽑아 그냥 대표팀을 만든다면 {A,B,C}와 {C,B,A}는 같은 팀이므로 조합입니다. 코드로 옮기면, 결과를 리스트로 담을 때 순서가 의미가 있으면 순열, 집합으로 담아도 손실이 없으면 조합입니다.

<u>**순열(Permutation)**</u>은 $n$개 중 $r$개를 골라 **순서대로 나열**하는 경우의 수입니다.

$$P(n, r) = n \times (n-1) \times \cdots \times (n-r+1) = \frac{n!}{(n-r)!}$$

> **해석** 유도는 단순합니다. 첫 자리에 올 수 있는 것은 $n$가지, 그 다음 자리는 이미 하나를 썼으므로 $n-1$가지, 이런 식으로 $r$개의 자리를 채우면 $n(n-1)\cdots(n-r+1)$입니다. 특히 $r = n$이면 $P(n,n) = n!$로, 전체를 한 줄로 세우는 경우의 수가 됩니다.

<u>**조합(Combination)**</u>은 $n$개 중 $r$개를 **순서 없이** 고르는 경우의 수입니다.

$$C(n, r) = \binom{n}{r} = \frac{P(n,r)}{r!} = \frac{n!}{r!\,(n-r)!}$$

> **해석** 조합은 순열에서 "중복 계산된 만큼 나누는" 방식으로 유도됩니다. 같은 $r$개를 골랐더라도 순열은 그것들을 나열하는 $r!$가지를 전부 별개로 셌으므로, 순서를 무시하려면 $r!$로 나누면 됩니다. 이 나눗셈이 정확히 떨어지는 것 자체가 조합론의 작지 않은 사실입니다.

| 구분 | 순서 | 중복 | 공식 | Python |
|---|---|---|---|---|
| 순열 | 있음 | 없음 | $\dfrac{n!}{(n-r)!}$ | `itertools.permutations` |
| 조합 | 없음 | 없음 | $\dfrac{n!}{r!(n-r)!}$ | `itertools.combinations` |
| 중복 순열 | 있음 | 있음 | $n^r$ | `itertools.product` |
| 중복 조합 | 없음 | 있음 | $\dbinom{n+r-1}{r}$ | `combinations_with_replacement` |

> **참고** 조합 기호 $\binom{n}{r}$을 <u>**이항계수(binomial coefficient)**</u>라 부르는 이유는 이 값이 $(x+y)^n$을 전개했을 때 $x^r y^{n-r}$ 항의 계수와 정확히 일치하기 때문입니다. $(x+y)^n$은 $(x+y)$를 $n$번 곱하는 것이고, 각 괄호에서 $x$와 $y$ 중 무엇을 고를지 정하는 문제이므로 "$n$개 중 $x$를 고를 $r$개를 정하는" 조합 문제와 같습니다.

## 2. 중복 순열과 중복 조합 — 같은 것을 다시 뽑아도 될 때

> **개요** 뽑은 것을 다시 넣고 뽑는 상황은 실무에서 오히려 더 흔합니다. 두 가지 중복 유형을 구분합니다.

<u>**중복 순열(Permutation with Repetition)**</u>은 매번 $n$가지 전부에서 고르는 것을 $r$번 반복하므로 $n^r$입니다. 각 자리가 서로 독립이라는 점이 핵심입니다.

$$\text{4자리 PIN 코드} = 10^4 = 10{,}000, \qquad \text{8자리 소문자 비밀번호} = 26^8 \approx 2 \times 10^{11}$$

<u>**중복 조합(Combination with Repetition)**</u>은 순서를 무시하면서 중복을 허용하는 경우로, 공식이 덜 직관적입니다.

$$H(n, r) = \binom{n + r - 1}{r}$$

> **해석** 이 공식은 <u>**막대와 별(stars and bars)**</u>이라는 기법으로 유도됩니다. "3종류의 아이스크림에서 5개를 고른다(같은 맛 여러 개 가능)"는 문제는 "별 5개를 3개의 칸에 나눠 담기"와 같고, 칸을 나누려면 막대 2개가 필요합니다. 그러면 별 5개와 막대 2개, 총 7개의 자리 중 막대가 놓일 2자리를 고르는 문제가 되어 $\binom{7}{2} = \binom{5+3-1}{5} = 21$입니다. 즉 $n$종류를 나누려면 막대가 $n-1$개 필요하므로 전체 자리가 $n+r-1$개가 됩니다.

> **팁** 중복 조합은 "총합이 정해진 음이 아닌 정수해의 개수"와 같은 문제입니다. $x_1 + x_2 + x_3 = 5$의 음이 아닌 정수해는 $H(3,5) = 21$개입니다. 자원 배분, 동전 거스름돈의 조합 수, DP 상태 개수 추정 등에서 이 형태가 자주 등장합니다.

## 3. 파스칼의 삼각형 — 점화식이 말하는 것

> **개요** $C(n,r) = C(n-1,r-1) + C(n-1,r)$은 단순한 계산 트릭이 아니라 조합론적 의미를 갖는 등식이며, 그 의미를 알면 DP 점화식을 세우는 감각으로 이어집니다.

![파스칼의 삼각형과 조합 점화식](https://raw.githubusercontent.com/SeongminJaden/dev-curriculum-assets/main/lectures/part19_이산수학과_비트_연산/19-3_순열과_조합/assets/diagram1_pascal_triangle.svg)

> **해석** 위 삼각형에서 각 수는 바로 위 두 수의 합입니다. 이것이 점화식 $C(n,r) = C(n-1,r-1) + C(n-1,r)$이며, 조합론적으로 읽으면 다음과 같습니다. $n$개 중 $r$개를 고를 때 **특정 원소 하나**(예: 첫 번째 원소)에 주목해 경우를 둘로 나눕니다. 그 원소를 **고른 경우**라면 남은 $n-1$개 중 $r-1$개를 더 골라야 하므로 $C(n-1, r-1)$가지, **고르지 않은 경우**라면 남은 $n-1$개 중 $r$개를 전부 골라야 하므로 $C(n-1, r)$가지입니다. 두 경우는 겹치지 않고 합치면 전체이므로 더하면 됩니다. "하나의 원소에 대해 포함/미포함으로 쪼갠다"는 이 발상은 배낭 문제를 비롯한 수많은 DP 점화식의 원형입니다.

```python
# 점화식을 그대로 DP로 옮기면 팩토리얼 없이 이항계수를 구할 수 있습니다
def binom(n, r):
    dp = [[0] * (n + 1) for _ in range(n + 1)]
    for i in range(n + 1):
        dp[i][0] = 1                                  # C(i,0) = 1
        for j in range(1, i + 1):
            dp[i][j] = dp[i-1][j-1] + dp[i-1][j]      # 파스칼 점화식
    return dp[n][r]
```

> **팁** 큰 $n$에 대해 $\frac{n!}{r!(n-r)!}$을 곧이곧대로 계산하면 중간 값인 $n!$이 폭발해 오버플로가 나거나 느려집니다. 파스칼 점화식을 쓰면 덧셈만으로 구할 수 있어 <u>**모듈러 산술(modular arithmetic)**</u>과도 잘 맞습니다. 경쟁 프로그래밍에서 $\binom{n}{r} \bmod p$를 구할 때 이 DP나 페르마 소정리를 이용한 역원 계산을 쓰는 이유입니다.

> **참고** 이 삼각형은 프랑스 수학자 블레즈 파스칼(Blaise Pascal)의 1654년 저작 *Traité du triangle arithmétique*로 서양에 널리 알려졌지만, 그보다 훨씬 앞서 11세기 페르시아의 알카라지(al-Karaji)와 13세기 중국의 양휘(楊輝)가 같은 구조를 기록했습니다. 중국에서는 지금도 이를 양휘 삼각형이라 부릅니다.

## 4. 비둘기집 원리 — 가장 단순하지만 강력한 논증

> **Q.** "비둘기가 집보다 많으면 한 집에 두 마리가 있다"는 게 너무 당연한데, 이게 왜 정리로 불릴 만한가요?
> **A.** 당연함이 곧 강력함이기 때문입니다. 이 원리는 **구체적으로 어느 집인지 몰라도** 존재를 증명해줍니다. 해시 충돌이 존재한다는 것을 증명할 때 우리는 어떤 두 문자열이 충돌하는지 전혀 모르지만, 입력이 출력보다 많다는 사실만으로 충돌의 존재를 확정할 수 있습니다. 이렇게 "찾지 않고도 있음을 아는" 논증 방식이 존재 증명의 대표적 도구입니다.

<u>**비둘기집 원리(Pigeonhole Principle)**</u>는 다음과 같이 진술됩니다.

$$n\text{개의 물건을 } k\text{개의 상자에 넣으면}, \ n > k \text{일 때 최소 한 상자에는 } \left\lceil \frac{n}{k} \right\rceil \text{개 이상이 들어간다}$$

| 상황 | 비둘기 | 집 | 결론 |
|---|---|---|---|
| 해시 테이블 | 가능한 모든 키 | 버킷 개수 | 충돌은 반드시 존재 |
| 무손실 압축 | 모든 $n$비트 파일 | 더 짧은 비트열 | 모든 파일을 줄이는 압축기는 불가능 |
| 32비트 정수 | 43억 개 초과 데이터 | $2^{32}$가지 값 | 중복 값이 반드시 존재 |
| 생일 문제 | 366명 | 365일 | 생일이 같은 두 사람 존재 |

> **참고** 무손실 압축에 대한 결론은 실무적으로 중요합니다. "모든 파일을 최소 1비트라도 줄이는 압축 알고리즘"은 원리적으로 존재할 수 없습니다 — $n$비트 파일은 $2^n$가지인데 $n$비트 미만의 비트열은 $2^n - 1$가지뿐이라 전단사가 불가능하기 때문입니다. 그래서 실제 압축기는 "자주 나오는 패턴을 짧게, 드문 패턴을 길게" 맞바꾸며, 이미 압축된 파일을 다시 압축하면 오히려 커지는 현상이 여기서 나옵니다.

> **팁** 비둘기집 원리는 알고리즘 설계에도 쓰입니다. "1부터 $n+1$까지의 수 중 $n+1$개를 골랐다면 서로 배수 관계인 쌍이 존재한다" 같은 문제는 적절한 "집"을 정의하는 순간 증명이 끝납니다. 문제에서 "반드시 존재함을 보여라"라는 표현이 나오면 비둘기집을 먼저 의심해보세요.

## 5. 경우의 수를 실행 시간으로 옮기기 — 실무 감각

> **개요** 이 강의의 핵심 실용 파트입니다. 경우의 수를 계산했다면, 그 숫자가 몇 초인지 즉시 환산할 수 있어야 합니다.

![경우의 수 증가율과 완전탐색 가능 한계](https://raw.githubusercontent.com/SeongminJaden/dev-curriculum-assets/main/lectures/part19_이산수학과_비트_연산/19-3_순열과_조합/assets/diagram2_growth_limits.svg)

> **해석** 위 그림은 $n$이 커질 때 $n^2$, $2^n$, $n!$이 어떤 속도로 벌어지는지를 실행 시간 기준선과 함께 보여줍니다. 현대 CPU가 단순 연산을 초당 약 $10^8$회 처리한다고 잡는 것이 실무의 관행적 기준선입니다. 이 기준으로 보면 $2^n$은 $n \approx 25$ 부근에서, $n!$은 $n \approx 11$ 부근에서 1초 선을 넘습니다. 두 곡선이 갈라지는 지점의 $n$ 값이 이렇게 작다는 것이 핵심입니다 — $n!$은 $n$이 겨우 두 자리만 되어도 손쓸 수 없어집니다.

| $n$ | $n^2$ | $2^n$ | $n!$ |
|---|---|---|---|
| 10 | 100 | 1,024 | 약 360만 |
| 15 | 225 | 32,768 | 약 1.3조 |
| 20 | 400 | 약 100만 | 약 $2.4 \times 10^{18}$ |
| 25 | 625 | 약 3,300만 | 계산 불가 |
| 30 | 900 | 약 10억 | 계산 불가 |

> **팁** 실무에서 쓰는 어림 기준은 다음과 같습니다.
> - $n \leq 10\text{~}11$: $n!$ 완전탐색(모든 순열 시도)이 가능합니다. 외판원 문제를 완전탐색으로 푸는 한계가 대략 여기입니다.
> - $n \leq 20\text{~}25$: $2^n$ 부분집합 완전탐색(비트마스크)이 가능합니다.
> - $n \leq 5{,}000$: $O(n^2)$ 이중 반복문이 가능합니다.
> - $n \leq 10^6$: $O(n \log n)$ 정렬 기반 접근이 필요합니다.
>
> 문제를 받으면 코드를 짜기 전에 $n$의 최대값을 먼저 확인하고 이 표에 대입해보세요. **$n$의 크기가 알고리즘을 역으로 알려줍니다** — $n \leq 20$이라면 출제자가 비트마스크를 의도했을 가능성이 높습니다.

> **참고** 이렇게 계산한 경우의 수를 실제로 탐색하는 알고리즘(백트래킹, 가지치기, 비트마스크 DP, 분기 한정법)의 구현은 Part 16에서 다룹니다. 이 강의의 목표는 "그 알고리즘을 써야 하는지, 아니면 애초에 다른 접근이 필요한지"를 **코드를 짜기 전에** 판단하는 것입니다.

## 요약

> **핵심**
> - **순열** $P(n,r) = \frac{n!}{(n-r)!}$은 순서가 의미를 갖는 선택, **조합** $C(n,r) = \frac{n!}{r!(n-r)!}$은 순서를 무시하는 선택입니다. 조합은 순열에서 중복 계산된 $r!$을 나눈 것입니다.
> - **중복 순열**은 $n^r$(각 자리가 독립), **중복 조합**은 $\binom{n+r-1}{r}$(막대와 별로 유도)이며, 후자는 "총합이 정해진 음이 아닌 정수해의 개수" 문제와 같습니다.
> - **파스칼 점화식** $C(n,r) = C(n-1,r-1) + C(n-1,r)$은 "특정 원소를 고르는가/고르지 않는가"로 경우를 쪼갠 결과이며, 이 발상이 수많은 DP 점화식의 원형입니다.
> - **비둘기집 원리**는 대상을 찾지 않고도 존재를 증명하며, 해시 충돌의 불가피성과 만능 무손실 압축의 불가능성을 설명합니다.
> - 초당 약 $10^8$ 연산을 기준선으로 잡으면 $n! $은 $n \approx 11$, $2^n$은 $n \approx 25$에서 1초를 넘습니다. **$n$의 최대값을 먼저 보고 알고리즘을 역추론하세요.**
