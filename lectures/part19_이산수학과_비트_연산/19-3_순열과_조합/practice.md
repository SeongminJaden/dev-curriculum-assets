# 19-3. 순열과 조합 — 실습

> **참고** 이 실습은 제출물이 아니라, 경우의 수를 직접 세어보고 그 숫자를 실행 시간 감각으로 옮겨보는 과정입니다.

## 실습 1 — 순열과 조합 직접 세어보기

- [ ] `[1,2,3,4,5]`에서 3개를 뽑는 순열과 조합을 `itertools.permutations`와 `itertools.combinations`로 각각 생성하고 개수를 세어보세요.
- [ ] 센 개수가 공식 $P(5,3)=60$, $C(5,3)=10$과 일치하는지 확인하고, 두 값의 비가 정확히 $3! = 6$인 이유를 한 문장으로 적어보세요.
- [ ] 같은 리스트로 중복 순열(`product(repeat=3)`)과 중복 조합(`combinations_with_replacement`)의 개수를 세고, $5^3 = 125$와 $\binom{7}{3} = 35$와 각각 맞는지 확인해보세요.

## 실습 2 — 파스칼 점화식으로 이항계수 구하기

- [ ] 파스칼의 삼각형 0행부터 10행까지를 2차원 리스트로 생성해 출력해보세요.
- [ ] $\binom{30}{15}$를 (a) 팩토리얼 공식으로, (b) 파스칼 점화식 DP로 각각 계산하고 두 값이 같은지 확인해보세요.
- [ ] $n$을 100, 500, 1000으로 키우며 두 방식의 실행 시간을 측정하고, 어느 쪽이 왜 더 유리한지 기록해보세요.

## 실습 3 — 비둘기집 원리 실측하기

- [ ] 임의의 문자열 10만 개를 만들어 `hash(s) % 1000`을 적용하고, 같은 값을 갖는 쌍이 존재하는지 확인해보세요.
- [ ] 버킷 수를 10만, 100만으로 늘려도 충돌이 사라지지 않는지 확인하고, 왜 사라질 수 없는지 비둘기집 원리로 설명해보세요.
- [ ] 이미 압축된 파일(예: `.zip` 또는 `.png`)을 다시 압축해 크기를 비교하고, 결과를 무손실 압축의 한계와 연결해 한 문단으로 적어보세요.

## 실습 4 — 경우의 수를 실행 시간으로 환산하기

- [ ] $n$을 8부터 12까지 바꿔가며 `permutations(range(n))`를 전부 순회하는 데 걸리는 시간을 측정해 표로 정리해보세요.
- [ ] $n$을 18부터 24까지 바꿔가며 `for mask in range(1 << n)` 순회 시간을 측정하고, 마찬가지로 표를 만들어보세요.
- [ ] 두 표에서 실행 시간이 1초를 넘는 $n$ 값을 각각 찾고, 강의의 기준($n! \approx 11$, $2^n \approx 25$)과 자신의 환경이 얼마나 차이 나는지 비교해보세요.

> **팁** 실습 4에서 순회 자체 외에 아무 연산도 넣지 않으면 실제 문제보다 훨씬 빠르게 측정됩니다. 각 경우마다 최소한의 검사(예: 합 계산)를 넣어보고 한계 $n$이 얼마나 앞당겨지는지 확인하면, 실전에서 여유를 얼마나 잡아야 하는지 감이 잡힙니다.
