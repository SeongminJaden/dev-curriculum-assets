# 19-1. 대수 법칙과 멱등성 — 실습

> **참고** 이 실습은 제출물이 아니라, 연산의 대수적 성질을 코드로 직접 확인하고 재시도 안전성을 손으로 검증하는 과정입니다.

## 실습 1 — 멱등성 판별하고 반례 만들기

- [ ] `abs(x)`, `sorted(list)`, `max(a, b)`를 각각 두 번 연속 적용하는 코드를 작성하고, 1회 적용 결과와 값이 같은지 `assert`로 확인해보세요.
- [ ] `counter += 1`과 `list.append(x)`를 두 번 실행해 결과가 달라지는 것을 출력으로 확인하고, 어떤 값이 얼마나 어긋나는지 기록해보세요.
- [ ] 비멱등 함수 `add_item(cart, item)`을 "이미 있으면 추가하지 않는" 형태로 고쳐 멱등하게 만들고, 3회 호출해도 장바구니 크기가 변하지 않음을 확인해보세요.

## 실습 2 — 멱등성 키로 POST를 안전하게 만들기

- [ ] 딕셔너리 하나를 저장소로 삼아, `key -> 응답`을 기록하는 `handle_request(idem_key, payload)` 함수를 작성해보세요.
- [ ] 같은 `idem_key`로 3번 호출했을 때 실제 처리는 1번만 일어나고 나머지는 저장된 응답이 반환되도록 구현하고, 처리 횟수를 카운터로 세어 확인해보세요.
- [ ] `idem_key`를 다르게 주면 새로 처리되는지 확인하고, 이 구조가 "결제 중복"을 어떻게 막는지 한 문장으로 적어보세요.

## 실습 3 — 결합법칙과 병렬 리듀스

- [ ] 정수 리스트 100만 개를 만들고, 왼쪽부터 순차로 접는 `reduce` 합산과 리스트를 4등분해 부분합을 낸 뒤 합치는 방식의 결과가 같은지 비교해보세요.
- [ ] 같은 실험을 뺄셈(`a - b`)으로 반복하고, 두 방식의 결과가 달라지는 것을 확인한 뒤 그 이유를 결합법칙으로 설명해보세요.
- [ ] `[1e20, -1e20, 1.0]` 같은 부동소수점 리스트에서 더하는 순서를 바꿔가며 합을 구하고, 결과가 달라지는 조합을 최소 하나 찾아 기록해보세요.

## 실습 4 — 교환·결합·멱등을 모두 만족하는 병합 만들기

- [ ] Python `set` 두 개를 만들어 `A | B`와 `B | A`, `(A|B)|C`와 `A|(B|C)`, `A|A`와 `A`가 각각 같은지 `assert`로 확인해보세요.
- [ ] 세 개의 "복제본" 집합을 만들고, 병합 순서를 무작위로 섞어 5가지 순열로 병합한 뒤 최종 결과가 모두 동일한지 확인해보세요.
- [ ] 같은 실험을 `set` 대신 `list`와 `+` 연산으로 반복하고, 왜 리스트로는 순서 무관 수렴이 되지 않는지 한 문장으로 적어보세요.

> **팁** 실습 3의 부동소수점 실험에서 반례를 찾기 어렵다면, 아주 큰 수와 아주 작은 수를 섞으세요. 크기 차이가 클수록 반올림으로 작은 값이 통째로 삼켜져 결합법칙이 눈에 띄게 깨집니다.
