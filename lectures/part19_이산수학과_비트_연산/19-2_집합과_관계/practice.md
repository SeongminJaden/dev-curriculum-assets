# 19-2. 집합과 관계 — 실습

> **참고** 이 실습은 제출물이 아니라, 집합 연산과 관계의 성질을 코드로 직접 확인하며 자료구조와 연결해보는 과정입니다.

## 실습 1 — 집합 연산과 드모르간 법칙 검증

- [ ] 두 집합 `A = {1,2,3,4,5}`, `B = {4,5,6,7}`을 만들고 합집합·교집합·차집합·대칭차를 각각 계산해 출력해보세요.
- [ ] 전체집합 `U = set(range(1,11))`을 정하고, `U - (A | B)`와 `(U - A) & (U - B)`가 같은 집합인지 `assert`로 확인해보세요.
- [ ] `if not (x > 10 or y < 0)` 형태의 조건문을 드모르간으로 변환해 다시 쓰고, `x`와 `y`에 각각 4가지 값을 넣어 두 형태의 결과가 모두 일치하는지 표로 정리해보세요.

## 실습 2 — 멱집합과 $2^n$ 직접 세어보기

- [ ] 원소 4개짜리 리스트에서 모든 부분집합을 재귀나 반복문으로 생성하고, 개수가 정확히 16개인지 확인해보세요.
- [ ] 같은 부분집합 목록을 `for mask in range(1 << n)` 방식으로 다시 생성하고, 두 결과가 동일한 집합족인지 비교해보세요.
- [ ] $n$을 10, 15, 20, 24로 늘려가며 `range(1 << n)` 순회에 걸리는 시간을 측정하고, $n$이 1 늘 때 시간이 몇 배가 되는지 계산해보세요.

## 실습 3 — 함수의 단사·전사 판별하기

- [ ] 정의역 `{0..9}`에서 `f(x) = x % 3`, `g(x) = x + 100`, `h(x) = x` 세 함수의 출력을 모두 나열해보세요.
- [ ] 각 함수가 단사인지 전사인지(공역을 명시하고) 판별하고, 판별 근거를 한 줄씩 적어보세요.
- [ ] 문자열 100만 개에 `hash(s) % 1000`을 적용해 버킷별 개수를 세고, 같은 버킷에 2개 이상 들어간 사례를 찾아 해시 함수가 단사가 아님을 실측으로 확인해보세요.

## 실습 4 — 동치 관계로 유니온 파인드 만들기

- [ ] 원소 10개에 대해 `parent` 배열을 초기화하고 `find(x)`와 `union(a, b)`를 직접 구현해보세요.
- [ ] `union(1,2)`, `union(2,3)`, `union(5,6)`을 수행한 뒤 `find(1) == find(3)`이 참인지 확인하고, 이것이 추이성의 결과임을 확인해보세요.
- [ ] 최종 상태에서 원소들을 동치류(그룹)별로 묶어 출력하고, 모든 원소가 정확히 하나의 그룹에만 속하는지(= 분할인지) 검증해보세요.

> **팁** 실습 4에서 `find`에 경로 압축(path compression)을 넣기 전과 후의 호출 깊이를 세어 비교해보세요. 동치류의 대표원소를 무엇으로 두든 그룹 구성 자체는 변하지 않는다는 점이 최적화를 안전하게 만드는 근거입니다.
