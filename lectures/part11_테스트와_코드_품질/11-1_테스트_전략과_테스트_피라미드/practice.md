# 11-1 실습 — 테스트 전략과 테스트 피라미드

> **참고** 제출물이 아니라, 오늘 배운 테스트 레벨 구분과 피라미드 개념을 직접 적용해 보는 실습입니다.

## 실습 1. 기능 하나를 세 레벨로 쪼개보기

- [ ] 최근에 사용한 앱이나 웹 서비스에서 기능을 하나 골라주세요. (예: 회원가입, 장바구니 담기, 검색)
- [ ] 그 기능을 검증할 단위 테스트 시나리오를 한 문장으로 적어보세요.
- [ ] 같은 기능을 검증할 통합 테스트 시나리오를 한 문장으로 적어보세요.
- [ ] 같은 기능을 검증할 E2E 테스트 시나리오를 한 문장으로 적어보세요.
- [ ] 세 시나리오가 서로 어떻게 다른 의존성을 포함하는지 비교해서 적어보세요.

## 실습 2. 아이스크림콘 판별하기

- [ ] 주변에서 들어봤거나 직접 겪은 프로젝트(학교 팀 프로젝트도 좋습니다) 하나를 떠올려보세요.
- [ ] 그 프로젝트에 테스트가 있었다면, 단위·통합·E2E 중 어떤 레벨이 가장 많았을지 추정해보세요.
- [ ] 만약 피라미드 모양이 아니라 아이스크림콘 모양이었다면, 어떤 증상이 있었을지 (또는 있었을 것 같은지) 적어보세요.

## 생각해볼 질문

- [ ] "테스트는 많을수록 무조건 좋다"는 말에 왜 동의하기 어려운지, 이번 강의 내용을 근거로 설명해보세요.
- [ ] 여러분이 만약 순수 계산 로직 위주의 라이브러리를 만든다면, 세 레벨 중 어디에 가장 많은 비중을 둬야 할지 이유와 함께 적어보세요.
