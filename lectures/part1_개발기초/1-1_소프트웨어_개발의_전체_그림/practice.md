# 1-1 실습 — 소프트웨어 개발의 전체 그림

> **참고** 제출물이 아니라, 오늘 배운 두 개의 지도(시간 축·계층 축)를 스스로 그려보는 실습입니다.

## 실습 1. 나만의 서비스로 SDLC 그려보기

- [ ] 최근에 사용한 앱이나 웹 서비스를 하나 골라주세요.
- [ ] 그 서비스에 있었을 법한 기능을 하나 정해주세요.
- [ ] 아래 6단계 각각에 어떤 일이 있었을지 한 문장씩 적어보세요.
  - [ ] 요구사항 정의
  - [ ] 설계
  - [ ] 구현
  - [ ] 빌드·테스트
  - [ ] 배포
  - [ ] 운영·유지보수

## 실습 2. 요청 처리 계층 따라가기

- [ ] 실습 1에서 고른 기능이 사용자 계층부터 하드웨어 계층까지 어떤 순서로 처리될지 정리해보세요.
- [ ] 각 계층에서 문제가 생겼을 때 나타날 수 있는 증상을 하나씩 상상해서 적어보세요. (예: 네트워크 계층 문제 → "요청이 아예 도달하지 않는다")
