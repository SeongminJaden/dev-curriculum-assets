# 1-3. 자료구조와 알고리즘 복잡도 — 실습

> **참고** 이 실습은 제출물이 아니라, Big-O 표기법과 자료구조별 복잡도를 직접 계산하고 비교해보는 과정입니다.

## 실습 1 — 코드의 시간복잡도 직접 도출하기

- [ ] 반복문이 하나만 있는 코드(예: 배열 전체를 한 번 순회하며 합을 구하는 코드)를 작성하고, 이 코드의 시간복잡도를 $O$ 표기로 적어보세요.
- [ ] 반복문 두 개를 중첩한 코드(예: 배열에서 모든 쌍을 비교하는 코드)를 작성하고, 시간복잡도를 도출해보세요.
- [ ] 정렬된 배열에서 이진 탐색을 직접 구현하고, 반복(또는 재귀)이 몇 번 만에 끝나는지 $n$ 값을 바꿔가며 세어보세요.

## 실습 2 — 자료구조 선택 비교

- [ ] 1,000개의 정수를 담은 배열과 연결 리스트를 각각 만들어보세요.
- [ ] 두 구조에서 "맨 앞에 새 값을 삽입하는" 연산을 수행하고, 어느 쪽이 더 적은 작업으로 끝나는지 직접 코드를 실행해 비교해보세요.
- [ ] 같은 배열과 연결 리스트에서 "가운데 위치의 값을 읽는" 연산을 비교하고, 실습 결과가 강의에서 배운 복잡도 표와 일치하는지 확인해보세요.

## 실습 3 — 해시 테이블 체감하기

- [ ] 10만 개의 문자열 데이터를 배열에 저장하고 특정 문자열을 순차 탐색으로 찾는 시간을 측정해보세요.
- [ ] 같은 데이터를 해시 테이블(딕셔너리/맵)에 저장하고 같은 문자열을 찾는 시간을 측정해보세요.
- [ ] 두 결과를 비교하고, 데이터 양을 100만 개로 늘렸을 때 격차가 어떻게 달라질지 예상해보세요.

## 실습 4 — 정렬 알고리즘의 실제 격차 확인하기

- [ ] $O(n^2)$ 정렬(버블 정렬 또는 삽입 정렬)과 $O(n \log n)$ 정렬(병합 정렬 또는 언어 표준 정렬 함수)을 각각 구현하거나 사용해보세요.
- [ ] 데이터 크기를 1,000개 → 10,000개 → 100,000개로 늘려가며 두 정렬의 실행 시간을 기록해보세요.
- [ ] 기록한 시간을 표로 정리하고, 데이터가 10배 늘어날 때 각 정렬의 실행 시간이 몇 배로 늘어나는지 계산해보세요.

> **팁** 실습 3, 4는 코드 실행 시간을 측정하는 것이 핵심이므로, 사용 언어의 시간 측정 기능(예: Python의 `time` 모듈)을 활용하면 됩니다. 측정값 자체보다 "데이터가 늘 때 시간이 어떤 비율로 늘어나는가"라는 추세를 관찰하는 데 집중하세요.
