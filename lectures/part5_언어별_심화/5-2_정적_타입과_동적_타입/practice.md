# 5-2. 정적 타입과 동적 타입 — 실습

> **참고** 제출물이 아니라, 정적/동적 타입의 검사 시점 차이를 직접 코드로 확인해보는 과정입니다. 아래 체크리스트를 순서대로 진행해보세요.

## 실습 1 — JavaScript 버그를 직접 재현하기

- [ ] 두 개의 숫자를 인자로 받아 더하는 함수를 JavaScript로 작성해보세요.
- [ ] 이 함수를 호출할 때 숫자 대신 문자열을 실수로 전달해보고, 어떤 값이 반환되는지 확인해보세요.
- [ ] 반환된 값이 왜 그렇게 나왔는지, 암묵적 타입 변환 관점에서 설명해보세요.

## 실습 2 — 같은 코드를 TypeScript로 옮겨보기

- [ ] 실습 1의 함수에 매개변수와 반환값 타입을 명시해 TypeScript로 다시 작성해보세요.
- [ ] 같은 방식으로 문자열을 실수로 전달하는 호출부를 작성하고, 컴파일러가 어떤 오류 메시지를 보여주는지 확인해보세요.
- [ ] 오류 메시지가 정확히 어느 줄, 어느 인자를 지목하는지 확인하고 캡처해보세요.

## 실습 3 — 타입 검사 시점 비교

- [ ] 실습 1(JavaScript)에서 오류가 발견된 시점과, 실습 2(TypeScript)에서 오류가 발견된 시점을 각각 한 문장으로 정리해보세요.
- [ ] 만약 이 함수가 사용자가 특정 버튼을 눌렀을 때만 실행되는 드문 코드 경로였다면, JavaScript 버전에서 이 버그가 얼마나 늦게 발견될 수 있었을지 설명해보세요.

## 생각해볼 질문

- [ ] Python의 타입 힌트(`def add(a: int, b: int) -> int:`)는 왜 TypeScript와 달리 실행을 막지 못하는지 설명해보세요.
- [ ] 덕 타이핑(duck typing)이 유용한 실제 상황을 하나 떠올려, 정적 타입 언어라면 이를 어떻게 구현해야 할지 비교해보세요.
