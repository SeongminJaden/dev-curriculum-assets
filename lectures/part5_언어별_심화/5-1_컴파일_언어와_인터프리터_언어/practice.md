# 5-1. 컴파일 언어와 인터프리터 언어 — 실습

> **참고** 제출물이 아니라, 오늘 배운 두 실행 경로(AOT 컴파일, 인터프리터, JIT)를 스스로 확인해보는 과정입니다. 아래 체크리스트를 순서대로 진행해보세요.

## 실습 1 — 같은 로직을 두 언어로 짜서 체감해보기

- [ ] 1부터 1000만까지 정수를 더하는 단순한 반복문을 Python으로 작성해보세요.
- [ ] 같은 로직을 C로 작성하고 컴파일해보세요(`gcc` 등 사용 가능한 컴파일러를 사용하세요).
- [ ] 두 프로그램의 실행 시간을 각각 측정해보세요(Python은 `time` 모듈, C는 `time` 명령어 등을 사용하세요).
- [ ] 측정한 시간 차이가 왜 발생했는지, 본문에서 배운 "VM의 명령어 해석 오버헤드" 개념으로 한 문단 설명해보세요.

## 실습 2 — 바이트코드 들여다보기

- [ ] Python의 `dis` 모듈을 사용해, 방금 작성한 반복문 함수의 바이트코드를 출력해보세요.
- [ ] 출력된 바이트코드 명령어(`LOAD_FAST`, `BINARY_ADD` 등) 중 세 개를 골라, 각각이 어떤 동작을 의미하는지 조사해 정리해보세요.
- [ ] 이 바이트코드가 왜 특정 CPU의 기계어가 아니라 "가상의 명령어 집합"인지 한 문장으로 설명해보세요.

## 실습 3 — JIT 체감하기

- [ ] PyPy(또는 Node.js의 V8, 어느 쪽이든 설치 가능한 환경)로 같은 반복 로직을 실행해보세요.
- [ ] CPython(순수 인터프리터)과 PyPy(JIT 포함)의 실행 시간을 비교해보세요.
- [ ] 두 결과의 차이가 왜 발생하는지, "프로파일링 → hot path 감지 → 네이티브 컴파일" 흐름으로 설명해보세요.

## 생각해볼 질문

- [ ] 만약 실행 시간이 1초도 안 되는 짧은 스크립트라면, JIT 컴파일이 오히려 손해가 될 수 있는 이유는 무엇일까요?
- [ ] NumPy 같은 라이브러리가 "Python 문법"으로 반복 연산을 짜도 빠른 이유를, 오늘 배운 AOT/인터프리터 개념으로 설명해보세요.
