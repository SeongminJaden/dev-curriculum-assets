1. **기준 정의** — 성취도 및 교과목은 [기준 스프레드시트](https://docs.google.com/spreadsheets/d/13z8ospFYZb4ApZN_Zfyk4NIcyW_VexWW2DkBlBVUuCk/edit?gid=0#gid=0)에 정의된 대로 진행한다.
2. **참고 기준** — [참고 Notion 페이지](https://app.notion.com/p/3968a651517c80afb87eeddda2d0112f?showMoveTo=true&saveParent=true)를 반드시 참고하여 작성한다.
3. **1차 작성** — 교안은 우선 Notion에 작성한다.
4. **작성 원칙 준수** — 아래 작성 원칙(2장)을 반드시 지킨다.

## 2. 작성 원칙

### 2.1 콘텐츠 보완

- **실생활 예시 추가** — 개념의 흐름이나 메커니즘을 설명할 때, 시각 자료와 함께 실생활 시나리오 예시를 추가한다. 왜 이 개념이 필요한지 이해하는 데 도움이 된다.
  - 예) ROS2의 핵심 아이디어 구조
- **토글 기능 활용** — 수강생이 선택적으로 학습할 심화 자료는 토글 안에 넣는다.

### 2.2 핵심 내용 강조

핵심 내용을 강조하면 수강생이 직관적으로 인지할 수 있다.

- **중요 내용** — 필수로 암기·기억해야 하는 내용은 **빨간색 배경 + 굵은 글씨**로 강조
- **개념 설명** — 개념의 핵심이 되는 부분은 **굵은 글씨 + 밑줄**로 강조
- **팁 정보** — 콜아웃 기능을 활용해 시각적으로 전달
- **여러 개념 설명** — 각 내용을 불렛포인트로 구분

예) "인지는 ~~이고, 제어는 ~~이다."는 다음과 같이 구분한다.
- 인지: ~~이다.
- 제어: ~~이다.

#### 2.2.1 콜아웃 사용 규칙 (1장 1강 적용 기준)

마크다운 자체에는 색상 개념이 없으므로, 아래처럼 **인용블록(`>`) + 굵은 라벨**로 콜아웃 유형을 구분해 작성한다. 노션으로 이관할 때 라벨에 맞춰 실제 콜아웃 색상(예: Q&A → 노란색)을 지정한다. 아이콘·이모지는 붙이지 않는다(2.5 원칙 참고).

- **상단 메시지 강조** — 강의 최상단(학습 목표, 출처 등)은 지금처럼 인용블록으로 배치하고, 핵심 키워드만 굵게 표시한다.
  ```
  > **학습 목표**
  > - ...
  ```
- **질문과 답변(Q&A) 형태** — 본문 중 "왜 ~일까요?"처럼 질문을 던지고 답하는 부분은 `**Q.**` / `**A.**` 라벨의 인용블록으로 묶는다. (노션 이관 시 **노란색 콜아웃**)
  ```
  > **Q.** 로봇은 왜 필요할까요?
  > **A.** 경제성·성능·안전성 때문입니다.
  ```
- **제목 섹션 하단 개념 개요** — `##` 섹션 제목 바로 아래에 그 절에서 다룰 내용을 1~2문장으로 요약하는 `**개요**` 인용블록을 추가한다.
  ```
  ## 2. 로봇은 왜 필요할까요?

  > **개요** 로봇 도입이 늘어나는 이유를 경제성·성능·안전성 세 가지 관점에서 살펴봅니다.
  ```
- **주의사항 / 참고 / 팁** — Q&A와 동일하게 콜아웃(인용블록) 기능을 사용하며, `**참고**` 또는 `**팁**` 라벨을 붙인다.
  ```
  > **참고** 이 수치는 하드웨어 사양에 따라 달라질 수 있습니다.
  ```
- 기존에 쓰던 `**해석**`(이미지 설명), `**핵심**`(핵심 요약), `**원칙**` 라벨도 같은 인용블록 규칙을 유지한다.

### 2.3 문체

- 교안 내 "왜 그런가(근거)"처럼 부자연스러운 표현은 "왜 그럴까요?"와 같이 자연스러운 문체로 변경한다.
- **실습 영역은 존댓말**을 사용한다.
  - As-is) …분석한 A4 1쪽 보고서를 제출하라.
  - To-be) …제출하세요.
- 이론 교안 안의 실습은 **"과제"가 아닌 "실습"**이라는 용어를 사용한다.

### 2.4 분량

- 간소화된 영역이 많으므로, 새로운 기술 교안을 추가하기보다 **각 페이지의 분량을 늘려 분할**하는 방식으로 보완한다.

### 2.5 클로드 사용 흔적 제거

협업용으로는 무관하나, 수강생 공유용에서는 클로드가 직접적으로 드러나는 요소를 대체한다.

- 아이콘 사용
- 장 구분 시 과도한 구분선·박스 사용
- 기타 AI 생성 티가 나는 표현

### 2.6 시각 자료 (필수)

- 설명에 필요한 **다이어그램·이미지·도식을 반드시 포함**한다. (**강의당 최소 2개**)

### 2.7 대학 수업 기준 정합성

- 모든 강의는 **대학 수업 내용과 동일한 수준**으로 작성한다.
- 교안 제작에 참고할 교재는 같은 폴더에 존재한다.


### 2.8 강의 내용

- 모든 강의는 왜 이걸 쓰는지, 학문적으로 접근하되 현업에서도 사용하는 이유를 논리적으로 설명해야 하며 수식과 공식이 필요할 경우 적극적으로 설명하며 도입해야한다.
